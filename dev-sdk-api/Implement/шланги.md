# План: Фаза 8 — AddRouteMode & Интерактивная трассировка кабелей

Этот план описывает создание режима прокладки и ручного редактирования 3D-трасс (кабелей). Мы объединим автоматический физический расчет траекторий с возможностью ручной доводки формы, подразделением сплайна на редактируемые узлы и масштабированием локального сечения жгута.

---

## 1. Структура Данных и Настройки

### Схема в `store` / `placementData`
Каждое ручное соединение в массиве `placementData.connections` расширяется следующими свойствами:
```typescript
interface Connection {
  from: string;          // Идентификатор исходного шарда
  from_socket: string;   // Имя исходного сокета
  to: string;            // Идентификатор целевого сокета
  to_socket: string;     // Имя целевого сокета
  manual?: boolean;      // Флаг ручного редактирования трассы (блокирует авто-роутер)
  
  // Координаты контрольных узлов: [x, y, z, radius_scale]
  // x, y, z — в вокселях (мировые координаты)
  // radius_scale — множитель толщины кабеля в данном узле (по умолчанию 1.0)
  control_points?: Array<[number, number, number, number]>; 
}
```

### Добавление настроек в `editorSettings`
Добавляем новый параметр:
* `cable_subdivision_step` — шаг сэмплирования авто-кривой при заморозке (в вокселях, диапазон `10` - `100`, дефолт `30`).
* Добавить поле ввода степпера для этого параметра во вкладку «Редактор» в `settings_panel.js`.

## 2. Архитектура Графического Сплетения (Three.js Visuals)

Для рендеринга кабелей с переменной толщиной стандартная `THREE.Line` или `THREE.TubeGeometry` не подходит, так как они строят трубу одинакового диаметра.

### Алгоритм трубы переменного сечения (`VariableRadiusTubeGeometry`)
Мы реализуем генератор геометрии, который строит меш (Mesh) на основе сплайна:
1. Создаем `THREE.CatmullRomCurve3` из массива `control_points`.
2. Разбиваем кривую на $N$ сегментов ($N \approx 60-100$).
3. Для каждой точки разбиения вычисляем **касательный вектор (tangent)**.
4. Строим **кадр Френе-Серре (Frenet-Serret Frame)**: вектора нормали (Normal) и бинормали (Binormal) для получения плоскости сечения.
5. Находим локальный радиус путем линейной интерполяции `radius_scale` между ближайшими контрольными точками.
6. Выстраиваем вершины по окружности:
   $$\vec{P}_{circle}(\theta) = \vec{P}_{curve} + R_{base} \cdot R_{scale} \cdot (\cos\theta \cdot \vec{N} + \sin\theta \cdot \vec{B})$$
7. Собираем индексы треугольников для формирования стенок трубы и торцевых заглушек.

### Вспомогательные элементы управления (Helpers)
При выборе кабеля для ручного редактирования спавнятся хелперы в `routeEditorGroup`:
* **Контрольные узлы**: Меши сфер (`THREE.SphereGeometry`) в позициях контрольных точек.
* **Рамка сечения (Cross-section Frame)**:
  * Плоский прямоугольный контур, повторяющий форму сокета (ширину и высоту сокета в пинах).
  * Выделен жирным зеленым цветом (эмиссионный зеленый материал или толстая линия).
  * Позиционируется в плоскости нормали-бинормали.
  * Направление нормали контура выравнивается по касательному вектору кабеля:
    ```javascript
    const tangent = curve.getTangentAt(u);
    helperFrame.quaternion.setFromUnitVectors(new THREE.Vector3(0, 0, 1), tangent);
    ```

## 3. Режимы Интерактивного Взаимодействия

### Сценарий создания кабеля (`AddRouteMode`)
1. **ЛКМ по Сокету A**:
   * Проверка: сокет не должен быть перегружен.
   * Фиксация начала. Создаем временный объект THREE.Line для отображения трассы к мыши.
2. **Перемещение (`onPointerMove`)**:
   * Вычисляем точку пересечения луча мыши с плоскостью текущего уровня (или snap-объектами).
   * Перестраиваем временный сплайн.
3. **ЛКМ по Сокету B**:
   * Проверка совместимости (разные шарды, тип портов).
   * Вызов авто-роутера (например, `routeAllCables`), сэмплирование результата с шагом `cable_subdivision_step`.
   * Сохранение новой связи в `placementData` с флагом `manual: true`.
   * Автоматический переход в `TranslateMode` с выделением новой трассы.

### Сценарий редактирования кабеля
* При клике на кабель активируется `routeEditorGroup`.
* Выделение сферы-узла вешает на нее `TransformControls`.
* **Перемещение узла**: Перемещение сферы с помощью `TransformControls` (в режиме `translate`).
  * Включена привязка (snap) к воксельной сетке по умолчанию согласно `snap_step` в настройках.
* **Масштабирование узла**:
  * Drag за прямоугольную рамку сечения (или переключение `TransformControls` в режим `scale` клавишей `R`).
  * Масштаб сферы `scale.x` транслируется в `radius_scale` для этой точки.
  * Геометрия трубы перестраивается на лету.
* При отпускании мыши (`pointerup` на `TransformControls`) новое состояние `control_points` записывается в историю.

## 4. Синхронизация с сервером и авто-компилятором

* **Bypass на бэкенде (`persistence.py`)**:
  * В методах парсинга и экспорта (`save_layout`) связи с `manual: true` сохраняют свои точки без изменений.
  * Роутер бэкенда при расчетах коллизий воспринимает меши ручных кабелей как статические препятствия, обходя их при прокладке новых автоматических кабелей.
