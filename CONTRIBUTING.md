# Contributing

Contributions are welcome, but this project has strict ownership and licensing rules.

## Contribution Terms

By opening a pull request, submitting a patch, or contributing documentation, specifications, designs, examples, or assets, you agree to [CLA.md](CLA.md).

If you do not agree, do not submit the contribution.

## What This Means

- Accepted contributions may be used under AGPL-3.0-only, commercial licenses, proprietary licenses, dual licenses, or other terms chosen by the Project Owner.
- You may be credited as a contributor, but you do not receive ownership, governance, trademark, revenue, release, or product rights.
- Pull requests may be accepted, edited, squashed, rejected, removed, or replaced at the Project Owner's discretion.

## Before Opening a Pull Request

- Keep the change focused.
- Follow the architecture and specifications in `docs/`.
- Do not include incompatible third-party code, copied snippets, leaked code, or unlicensed assets.
- Clearly mention any external source, generated content, dataset, model, asset, or license involved.
- Run the relevant checks for the changed area when practical.

## Larger Changes

For large architectural changes, new subsystems, public APIs, file formats, or editor workflows, open an issue first. The Project Owner may require a separate signed assignment before merging substantial work.
