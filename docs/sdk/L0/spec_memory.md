## §5. Доменная Логика

Модуль `axipy.memory` отвечает за кроссплатформенное управление разделяемой памятью (Shared Memory) и организацию прямого доступа (через системные механизмы `mmap`) к структурам состояния и отладки Rust-ускорителя.

Изоляция платформозависимого системного кода решает проблему переносимости СДК. Разделяемая память в Linux (POSIX SHM через виртуальную файловую систему `/dev/shm`) и Windows (файловые маппинги через системные вызовы Win32 API) реализуется принципиально по-разному. Чтобы защитить прикладные слои SDK от вызовов Win32, библиотек `msvcrt` или Unix-специфичных флагов системного вызова `mmap`, вся системная работа по захвату, отображению, когерентной синхронизации и закрытию сегментов разделяемой памяти изолирована в Слое 0.

Доменная задача модуля — обеспечить сверхбыстрый межпроцессный обмен данными (IPC) по принципу Zero-Copy (без копирования в куче). Модуль преобразует сырые байтовые буферы ОС в строго типизированные представления `memoryview`, позволяя Python-скриптам в реальном времени считывать мембранные потенциалы нейронов, инъецировать токи через отладчик электрофизиологии (`shard.ephys`) и координировать Night Phase симуляции без накладных расходов на сериализацию и сетевые сокеты.
