# Реестр Системных Инвариантов, Граничных Случаев и Угроз (TROUBLE.md)

Единый реестр архитектурных и межкрейтовых ограничений системы Axicor Enterprise. Этот документ служит центральным источником правды для команды разработки, систем тестирования и системных промптов ИИ-помощников.

---

## 1. Семантика и Правила Нумерации

Для четкого разграничения типов угроз и уровней их влияния в проекте принята единая система префиксов и правил нумерации:

### Типы и префиксы записей
1. **`E-{NNN}` (Edge Cases)**: Глобальные граничные случаи, определенные на этапе проектирования архитектуры.
2. **`R-{NNN}` (Race Conditions)**: Глобальные потенциальные состояния гонки и коллизии конкурентного выполнения.
3. **`D-{NNN}` (Degradations)**: Возможные сценарии деградации производительности или отказоустойчивости системы.
4. **`INV-CROSS-{NNN}`**: Межкрейтовые инварианты — строгие соглашения между несколькими компонентами (крейтами), требующие взаимного соблюдения.
5. **`INV-{CRATE}-{NNN}`**: Локальные инварианты конкретного крейта (например, `INV-IPC-001`), описывающие внутренние ограничения логики или данных.

### Локальные расширения в ходе разработки
В процессе кодирования неизбежно выявляются новые локальные граничные случаи и угрозы. Чтобы не нарушать сквозную глобальную нумерацию, новые локальные инварианты кодируются с указанием типа внутри префикса крейта:
- `INV-{CRATE}-E{NNN}` — локальный граничный случай (например, `INV-IPC-E001`).
- `INV-{CRATE}-R{NNN}` — локальное состояние гонки (например, `INV-IPC-R001`).
- `INV-{CRATE}-D{NNN}` — локальная угроза деградации (например, `INV-IPC-D001`).

## 3. Глобальный Реестр Граничных Случаев (Edge Cases: E-NNN)

Сводная таблица распределения диапазонов глобальных граничных случаев по компонентам системы:

| ID | Компонент (Крейт) | Семантика |
| :--- | :--- | :--- |
| **<nobr>[E-001](specs/spec_L0/types_spec.md#L307)</nobr>** | `types` | **Bit Bleed (Кровотечение битов): Попытка упаковать координату `X`, превышающую 10 бит (например, 2048), в `PackedPosition`.**: Игнорируется старший бит, применяется жесткая маска `& 0x3FF`. Координата закольцовывается (Wrap-around), предотвращая затирание соседних полей `Y` и `Z`. |
| **<nobr>[E-002](specs/spec_L0/types_spec.md#L308)</nobr>** | `types` | **Иллюзия нулевой связи (Zero-Index Trap): Упаковка связи с аксоном `axon_id = 0` и `segment = 0` в `PackedTarget`.**: Формула выполняет сдвиг `(axon_id + 1)`, записывая в память `1`. Гарантируется, что бинарный `0` в памяти всегда обозначает пустоту (`None`), а не нулевой индекс. |
| **<nobr>[E-003](specs/spec_L0/types_spec.md#L309)</nobr>** | `types` | **Переполнение Burst-счетчика: Добавление 8-го спайка в 3-битный счетчик серий `SomaFlags` (лимит 0..7).**: Значение фиксируется на `7` (Saturating Clamp). Математика мутатора: `count.min(7)`. Переполнение в `0` аппаратно блокируется, чтобы не сломать кривую BDP-пластичности. |
| **<nobr>[E-004](specs/spec_L0/types_spec.md#L310)</nobr>** | `types` | **Нестандартная энтропия: Передача пустой строки `""` или строки с non-ASCII символами в `MasterSeed::from_str`.**: Функция `fnv1a` безопасно обрабатывает любые байтовые срезы без паник, возвращая стабильный детерминированный хэш (исключение паник при парсинге). |
| **<nobr>[E-005](specs/spec_L0/types_spec.md#L311)</nobr>** | `types` | **Float Bound Exclusivity: Максимальное теоретическое значение, возвращаемое `random_f32`.**: Строго ограничено `0.99999994` (за счет битовой маски `0x3F800000`). Гарантируется, что результат никогда не будет равен `1.0`, предотвращая `OutOfBounds` при масштабировании массивов в вышестоящих слоях. |
| **<nobr>[E-006](specs/spec_L0/types_spec.md#L312)</nobr>** | `types` | **Axon ID Overflow (Крах лимита аксонов): Превышение общего числа аксонов в шарде (`total_axons = Local + Virtual + Ghost`) значения `16 777 214` (лимит 24 бит поля `Axon_ID` с учетом сдвига).**: Математика упаковки `PackedTarget` остается слепой (без branch-проверок) ради O(1). Защита обеспечивается формированием жесткого инварианта для вышестоящих слоев: функция аллокации шарда обязана проверять `assert!(total_axons <= 0x00FFFFFE)`. Превышение прерывает компиляцию/загрузку с требованием перебалансировки шарда. |
| **<nobr>[E-007](specs/spec_L0/physics_spec.md#L755)</nobr>** | `physics` | **Зомби-спайки (Overjump): Быстрый аксон (`v_seg > 1`) математически перепрыгивает `AXON_SENTINEL` (`0x80000000`).**: Аппаратная блокировка через XOR-ловушку (`(h ^ AXON_SENTINEL) >= v_seg`). Сигнал замораживается на месте, предотвращая эпилепсию сети. |
| **<nobr>[E-008](specs/spec_L0/physics_spec.md#L756)</nobr>** | `physics` | **Отрицательная депрессия (Dale's Law Guard): Ослабление (LTD) превышает текущую массу синапса (`abs_w + delta < 0`).**: Масса аппаратно усекается до 0 (превращаясь в "немый" синапс — Silent Synapse), но биологический знак синапса сохраняется. Инверсия знака невозможна. |
| **<nobr>[E-009](specs/spec_L0/physics_spec.md#L757)</nobr>** | `physics` | **Темпоральный парадокс (Spike Birth): Инициализация головы нового спайка.**: Инициализируется через `0u32.wrapping_sub(v_seg)`.
