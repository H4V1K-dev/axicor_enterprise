# Experiment 2026-07-06: Growth v2 Parameter Sweep & Pruning Policy v0.3

## Overview
This experiment builds upon the Growth v2 Biology-Aligned Multifield Prototype (v0.2) to identify a low-pressure compile candidate ready for AOT-to-Flat runtime parity verification. Config 16 is not yet a final functional replay topology because the strict capture radius removes the `L4_spiny -> L5_spiny` projection.

Instead of simple discrete heuristics, we parameterize and sweep over:
- Pruning methods (uniqueness vs. softmax stochastic selection)
- Branching configurations (number of arbors and lengths)
- Guidance force parameters (fasciculation, repulsion, noise)
- Capture bounds (dendrite radius overrides)

## Directory Structure
- `AxiEngine/crates/test-harness/tests/baker_growth_v2_pruning_sweep.rs`: The integration test execution sweep.
- `reports/growth_v2_pruning_sweep_v0_3.md`: The detailed final scientific report evaluating the 16 configs.
- `scripts/plot_growth_v2_pruning_sweep.py`: Python script generating metrics visuals.
- `artifacts/`: Raw JSON generated by the Rust sweep runner. These files are local research artifacts and should not be treated as source.
- `images/`:
  - `3d_shard_overview.png`: 3D view of somas and axons showing terminal branching.
  - `3d_touch_points.png`: 3D view of accepted synapses color-coded by target type.
  - `projection_heatmap.png`: Synapse count matrix between layer pairs.
  - `fan_in_histogram.png`: Synapse density distribution per target soma.
  - `pruning_breakdown.png`: Log-scale bar chart of raw vs. accepted vs. uniqueness/cap pruned across configs.
  - `tki_spread_scatter.png`: Scatter plot comparing TKI vs. Spread Radius.
  - `pareto_success_vs_pressure.png`: Pareto frontier mapping Virtual->L4 success vs. target saturation count.

## Quick Start
Run the simulation sweep:
```bash
cargo test -p test-harness --features "cpu mvp-cpu-replay baker-probe" --test baker_growth_v2_pruning_sweep run_growth_v2_pruning_sweep -- --nocapture
```

Generate plots:
```bash
.venv/bin/python3 docs/engine/research/archive/2026-07-06_growth_v2_pruning_sweep_v0_3/scripts/plot_growth_v2_pruning_sweep.py
```
