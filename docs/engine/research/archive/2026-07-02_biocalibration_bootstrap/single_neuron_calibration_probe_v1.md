# Калибровка одиночного GLIF-нейрона (Single Neuron Physics Calibration Probe)
*(single-neuron-physics-calibration-probe-v1)*

Этот отчет представляет результаты симуляции одиночной GLIF-мембраны AxiEngine в сравнении с очищенными экспериментальными данными из `biological_calibration_pack_v1.json`. Калибровка проведена методом перебора сеток параметров в рамках изолированного тестового окружения Rust (`test-harness` под флагом `single-neuron-calibration`), без изменения констант самого движка.

## 2. Анализ соответствия GLIF-физики AxiEngine

### Метрики, которые текущий GLIF может воспроизводить:
1. **Рефрактерный период (refractory_period)**:
   - Отлично отражает разницу между быстроразряжающимися интернейронами (PV interneurons, оптимальный `refractory_period = 2` ms) и медленными возбуждающими пирамидами/SST-клетками (`refractory_period = 12..20` ms).
2. **Наклон f-I кривой на средних частотах (Slope)**:
   - Множитель `current_scale` в диапазоне `0.01 - 0.02` позволяет сжать биологический диапазон токов (0..350 pA) в целочисленный шаг интегрирования AxiEngine (0..7 единиц заряда за тик), обеспечивая адекватный прирост частоты спайков.
3. **Качественная утечка мембраны (`leak_shift`)**:
   - Масштабирует скорость возврата к покою ($V_{\text{rest}}$). PV-интернейроны со сверхнизким временем мембраны ($\tau_m = 22$ ms) требуют малого сдвига (`leak_shift = 2`), а медленные Scnn1a ($\tau_m = 26$ ms) требуют более выраженного сопротивления утечки (`leak_shift = 6`).

### Метрики, которые НЕ могут быть воспроизведены напрямую:
1. **Точная реобаза (Rheobase threshold)**:
   - В биологических нейронах переход к спайкам происходит плавно за счет субпороговых натриевых и калиевых токов. В точечном GLIF без плавающих каналов порог реобазы имеет дискретную погрешность из-за грубого округления токов (`current_scale`). Погрешность реобазы у PV и SST составила 30-60 pA.
2. **Адаптация частоты разрядов (Spike-Frequency Adaptation)**:
   - Текущий GLIF в AxiEngine при нулевом homeostasis_penalty (простая LIF-калибровка) выдает линейно-регулярные спайки при постоянном токе. Биологические клетки при этом демонстрируют экспоненциальное удлинение интервалов между спайками (ISI).

## 3. Области расширения физики AxiEngine

Для преодоления ограничений probe v1 рекомендуется внедрить следующие изменения:
1. **Включение адаптивного порога утечки (Adaptive Leak)**:
   - В коде `update_glif_voltage` уже заложены параметры `adaptive_leak_gain` и `adaptive_mode`. Их активация позволит динамически увеличивать проводимость утечки при накоплении гомеостатического сдвига порога, что симулирует калиевые токи AHP (послеспайковой гиперполяризации) и обеспечит естественную адаптацию частоты спайков.
2. **Субмиллисекундное интегрирование (Sub-millisecond Ticking)**:
   - Разрешение 1 tick = 1 ms слишком грубое для быстроразряжающихся клеток (PV могут выдавать спайки с частотой > 300 Hz, что требует шага в 2-3 ms на спайк, упираясь в предел Найквиста при шаге 1 ms). Переход на шаг 0.1 ms или 0.5 ms повысит точность расчета f-I кривых.

## 4. Пересчет legacy-параметров

Данные из Allen Calibration Pack могут использоваться для прямой автоматической перенастройки старых полей конфигурации:
* **`heartbeat_m`** (spontaneous period): Ранее задавался гипотетически. Теперь, на основе подтвержденных спонтанных ephys-свойств, для всех PV, SST, Cux2 и Scnn1a значение должно быть приравнено к 0 (отсутствие спонтанного водителя ритма), так как статус `inconclusive/checked_no_spikes` подтвердил отсутствие спайков при 0 pA.
* **`leak_shift`**: Может рассчитываться напрямую как логарифмическая аппроксимация времени жизни мембраны:
  $$\text{leak\_shift} = \text{round}(\log_2(\tau_m))$$
  Для $\tau_m = 13.24$ ms (Cux2) это дает `leak_shift = 4` (в симуляции лучше подошло `6`).
* **`refractory_period`**: Пересчитывается из биологического $AP_{\text{half-width}}$ с фиксированным запасом на фазу восстановления (обычно $3 \times AP_{\text{half-width}}$).
