# Аудит и консолидация кандидатов в биологические эталонные нейроны
*(reference-neuron-audit-v1)*

Этот отчет содержит объединенный аудит кандидатов в биологические эталонные нейроны (reference neurons), предложенных в ходе анализа первичных данных Allen Cell Types Database и отчетов языковых моделей. 

**Важное примечание по интеграции**: На текущем этапе ни один из этих нейронов **не вносится** в основную конфигурацию вычислительного ядра AxiEngine как безусловная «истина» (production baseline). Все перечисленные specimen ID рассматриваются исключительно в качестве **кандидатов** для внешнего тестирования и калибровки.

---

## 1. Шкала статусов доказательности

Для классификации степени достоверности и готовности кандидатов используется следующая шкала статусов:
- **Seed** — приоритетный первичный кандидат (эталон), который планируется извлечь и проверить в первую очередь для формирования базовой сетки.
- **Candidate** — потенциально ценный кандидат, требующий дополнительной программной проверки.
- **Contested** — спорный кандидат, имеющий классификационные расхождения, противоречия в метаданных или недоказанные свойства.
- **Backup** — вспомогательный/резервный кандидат, находящийся вне основного ядра калибровки.
- **Excluded-for-baseline** — исключен из основного эталонного набора (теоретические конструкты или крайние случаи, используемые только в стресс-тестах).

## 3. Анализ спорных кандидатов и расхождений

1. **Проверка Pacemaker-роли (313862134 vs 486754703)**:
   - Спецификация 313862134 (Sst) и ElecFeX-профиль 486754703 предложены как спонтанно активные клетки. Однако в корковых срезах in vitro SST-клетки демонстрируют pacemaking только в специфических условиях. Роль пейсмейкера (pacemaker) **не считается доказанной** до проведения детального скриптового анализа нулевых разверток (0 pA sweeps) для обоих нейронов.
   - *Решение*: Выполнить скриптовый анализ разверток без инжекции тока (0 pA sweeps), чтобы зафиксировать наличие или отсутствие спонтанной генерации спайков на сырых экспериментальных данных NWB.

2. **Classification Conflict в Allen Database (469801138)**:
   - specimen_id 469801138 имеет статус классификационного конфликта (classification conflict): в плоской ephys-таблице он указан как возбуждающий (excitatory), хотя Cre-линия Pvalb-IRES-Cre и aspiny морфология свидетельствуют о тормозном PV-профиле.
   - *Решение*: Не называть это ошибкой базы данных окончательно, пока скрипт не выполнит автоматическое извлечение и кросс-проверку Cre-линии, морфологии и типа дендритов (spiny/aspiny). Использовать этот ID как стресс-тест для классификатора.

3. **Верификация существования 313861411**:
   - DeepSeek в своем отчете объявил specimen_id 313861411 не существующим в базе. Этот ID присутствует в Allen Cell Types Database и относится к PV-клетке VISpm L4.
   - *Решение*: Оставить кандидата в списке аудита; программно извлечь его параметры скриптом сбора данных.

4. **Кросс-региональные кандидаты (321906005, 324257146)**:
   - Нейроны 321906005 (VISpm) и 324257146 (VISam) получены из зрительных зон, отличных от первичной зрительной коры (VISp). Физиология клеток в этих областях может иметь свою специфику.
   - *Решение*: Перевести эти нейроны в статус `Backup`. Приоритетным baseline для калибровки остаются строго VISp-нейроны.

## 4. План скриптового сбора данных (Allen SDK / API)

Для перехода к следующему этапу (сбору данных) сформирован перечень полей, которые необходимо извлечь из Allen Brain Map API/SDK для каждого утвержденного Specimen ID:

1. **Пассивные мембранные параметры**:
   - Потенциал покоя ($V_{\text{rest}}$, `vrest` или `v_baseline` в sweeps).
   - Входное сопротивление ($R_{\text{in}}$, `ri` или `input_resistance`).
   - Мембранная постоянная времени ($\tau_m$, `tau`).
   - Мембранная емкость ($C_m$, вычисляется как $\tau_m / R_{\text{in}}$).

2. **Характеристики возбудимости**:
   - Ток реобазы (`threshold_i_long_square`).
   - Порог генерации спайка ($V_{\text{threshold}}$, значение потенциала в точке dV/dt = 5% max(dV/dt) на первом спайке при реобазе).
   - Отношение скоростей нарастания/спада спайка (`upstroke_downstroke_ratio`).
   - Ширина полувысоты спайка (`action_potential_half_width`).

3. **Динамика отклика (long square sweeps)**:
   - Средняя частота генерации на токе (rheobase + 40..60 pA).
   - Индекс адаптации межиспайковых интервалов (`adaptation_index`).

4. **Спонтанная активность**:
   - Наличие спайков в развертках без инжекции тока (0 pA sweeps).

5. **Статус моделей**:
   - Проверка наличия GLIF-конфигураций (`m__glif` и `NeuronalModel` с фильтром `*LIF*`).
   - Проверка наличия ModelDB biophysical models.

Все извлеченные данные будут аккумулироваться в локальном кэше, сохраняя статус кандидатов без изменения рабочих файлов симулятора AxiEngine.
