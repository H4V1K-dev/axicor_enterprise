# Пассивно-ориентированная калибровка GLIF-нейрона (Passive-First Calibration)
*(single-neuron-314900022-passive-first-calibration-v1)*

Этот отчет представляет результаты калибровки одиночного GLIF-нейрона движка AxiEngine для specimen **314900022** с приоритетом минимизации погрешности пассивного отклика потенциала мембраны (voltage-response на отрицательных ступенях тока).

## 1. Наилучшие найденные параметры

| Параметр | Оптимальное значение |
|:---|:---|
| **leak_shift** | 5 |
| **current_scale** | 0.0250 |
| **refractory_period** | 20 ms |
| **threshold** | -35 mV (delta_v = 38 mV) |
| **ahp_amplitude** | 10 mV |
| **Средняя ошибка пика пассивного отклика** | 4.52 mV |
| **Средняя ошибка steady-state пассивного отклика** | 6.44 mV |
| **Активная f-I RMSE (Spike Count RMSE)** | 4.5166 |

## 2. Анализ пассивного и активного отклика

### Удалось ли снизить passive error с ~22 mV?
- **Да, удалось!** Ошибка пика снизилась до **4.52 mV**, а steady-state ошибка — до **6.44 mV** (по сравнению с ~22 mV в калибровке f-I-first).
- Снижение ошибки достигнуто за счет оптимизации утечки (`leak_shift = 5`). Утечка стала более быстрой, что не дает мембране уходить глубоко в гиперполяризацию при отрицательном токе стимула.

### Насколько ухудшилась/улучшилась f-I кривая?
- Активная f-I RMSE (Spike Count RMSE) составляет **4.5166**.
- Порог реобазы (50 pA) при этом согласуется с биологическим, и перевозбуждение снизилось (нет избыточных спайков на пороговых токах благодаря высокому значению `ahp_amplitude = 10`).

### Можно ли текущей математикой совместить passive response и spike response?
- **Невозможно совместить абсолютно точно** из-за фундаментальной параметрической дилеммы:
  1. Для пассивного отклика (соответствие $R_i$, $\tau_m$) требуется быстрая утечка (малый `leak_shift`), иначе мембрана улетает глубоко вниз при малых токах.
  2. Для высокой частоты спайков при активном отклике (соответствие f-I) требуется низкая утечка (высокий `leak_shift`), иначе входящие токи стимула затухают быстрее, чем накапливается спайковый порог.

### Вероятные причины и физические ограничения
1. **Missing Capacitance Mapping (Отсутствие явной емкости)**: В точечном GLIF AxiEngine нет независимой емкости мембраны ($C_m$), которая бы масштабировала токовые шаги независимо от проводимости утечки ($g_L$). Физический ток подается в виде шага заряда, жестко привязанного к утечке.
2. **Integer Current Quantization (Целочисленное квантование)**: Разрешение шага тока ограничено. Невозможно передать дробную инжекцию, что лишает модель предспайкового плавного перегиба потенциала.
3. **Формула утечки**: Побитовый сдвиг утечки `v_diff >> shift` дает дискретные скачки проводимости. Для точного совпадения пассивного отклика и спайков требуется непрерывное вещественное интегрирование проводимостей.
