## §3. Инварианты

Крейт `wire` гарантирует соблюдение 3 фундаментальных инвариантов, обеспечивающих сетевую безопасность, отсутствие Silent Data Corruption и строгую синхронизацию с SDK.

### §3.1. Структурные инварианты

- **INV-WIRE-001**: C-ABI совместимость и строгий размер структур. Все публичные структуры обязаны иметь атрибут `#[repr(C)]`, а их размеры должны быть жестко зафиксированы.
  - *Обоснование*: Любая неопределенность в выравнивании полей (struct padding) компилятором Rust приведет к Silent Data Corruption при обмене данными между GPU-ядрами, CPU-хостом и внешними клиентами (Python SDK).
  - *Следствие нарушения*: Смещение байтов, чтение мусора вместо валидных флагов, падение рантайма или непредсказуемое поведение симуляции.
  - *Где проверяется*: compile-time assert внутри `lib.rs` (константный блок проверок `const _: () = { assert!(...) };`).

- **INV-WIRE-002**: Безопасный zero-copy cast. Все сетевые заголовки должны реализовывать трейты `bytemuck::Pod` и `bytemuck::Zeroable` через макрос `decl_wire!`.
  - *Обоснование*: Любая промежуточная копия памяти или аллокация в горячем цикле сетевой трансляции спайков снижает пропускную способность шины и увеличивает p99 латентность.
  - *Следствие нарушения*: Снижение производительности сетевого стека, аллокации на куче в критическом пути исполнения.
  - *Где проверяется*: compile-time автопроверка компилятором через derive-макросы `bytemuck` в `decl_wire!`.

### §3.2. Семантические инварианты

- **INV-WIRE-003**: Строгая бинарная синхронизация с SDK. Генератор `build.rs` обязан автоматически транслировать бинарный макет всех `#[repr(C)]` структур в Python-модуль `contract_generated.py` при любой сборке крейта.
  - *Обоснование*: Предотвращает человеческий фактор и ошибки ручного копирования бинарных смещений полей в SDK на стороне Python.
  - *Следствие нарушения*: Рассинхронизация полей, Silent Data Corruption при парсинге отчетов, телеметрии или файлов внешними инструментами анализа.
  - *Где проверяется*: Автоматическая компиляция и проверка генерации в `build.rs` при сборке проекта.

## §4. Публичный API

### §4.1. Типы (Types)

Все типы крейта `wire` реализуют трейты `bytemuck::Pod` и `bytemuck::Zeroable` для безопасного O(1) приведения из байтовых буферов (`&[u8]`).

#### Домен 1: Fast-Path UDP (Spike Sync)
*   **`SpikeBatchHeaderV2`**
    *   **Размер / Выравнивание**: 16 байт / `align(4)`.
    *   **Поля**: `src_zone_hash` (u32), `dst_zone_hash` (u32), `epoch` (u32), `chunk_idx` (u16), `total_chunks` (u16).
    *   **Семантика**: Заголовок L7-фрагментированного батча спайков по UDP.
*   **`SpikeEventV2`**
    *   **Размер / Выравнивание**: 8 байт / `align(4)`.
    *   **Поля**: `ghost_id` (u32 — прямой индекс в VRAM получателя), `tick_offset` (u32).
    *   **Семантика**: Единичный спайк внутри батча. Массив этих структур следует сразу за заголовком.

#### Домен 2: Внешний I/O (Сенсоры, Моторы, R-STDP)
*   **`ExternalIoHeader`**
    *   **Размер / Выравнивание**: 20 байт / `align(4)`.
    *   **Поля**: `magic` ([u8; 4] = `"GSIO"` или `"GSOO"`), `zone_hash` (u32), `matrix_hash` (u32), `payload_size` (u32), `global_reward` (i16), `_padding` (u16).
    *   **Семантика**: Заголовок пакета внешней среды (сенсоры/моторы).
*   **`ControlPacket`**
    *   **Размер / Выравнивание**: 8 байт / `align(8)`.
    *   **Поля**: `magic` ([u8; 4] = `"DOPA"`), `dopamine` (i16), `_pad` (u16).
    *   **Семантика**: Пакет инъекции глобального дофамина (R-STDP Control).

#### Домен 3: Slow-Path TCP & Routing (Межзональная Топология)
*   **`GhostConnection`**
    *   **Размер / Выравнивание**: 8 байт / `align(4)`.
    *   **Поля**: `src_soma_id` (u32), `target_ghost_id` (u32).
    *   **Семантика**: Связь между сомой-источником и Ghost-аксоном получателя.
*   **`AxonHandoverEvent`**
    *   **Размер / Выравнивание**: 20 байт / `align(4)`.
    *   **Поля**: `origin_zone_hash` (u32), `local_axon_id` (u32), `entry_x` (u16), `entry_y` (u16), `vector_x` (i8), `vector_y` (i8), `vector_z` (i8), `type_mask` (u8), `remaining_length` (u16), `entry_z` (u8), `_padding` (u8).
    *   **Семантика**: Событие динамической передачи растущего аксона при пересечении границы шарда (Night Phase).
*   **`AxonHandoverPrune`**
    *   **Размер / Выравнивание**: 12 байт / `align(4)`.
    *   **Поля**: `target_zone_hash` (u32), `receiver_zone_hash` (u32), `dst_ghost_id` (u32).
    *   **Семантика**: Команда удаления Ghost-связи.
*   **`AxonHandoverAck`**
    *   **Размер / Выравнивание**: 16 байт / `align(4)`.
    *   **Поля**: `target_zone_hash` (u32), `receiver_zone_hash` (u32), `src_axon_id` (u32), `dst_ghost_id` (u32).
    *   **Семантика**: Подтверждение создания Ghost-аксона.

#### Домен 4: Телеметрия (WebSocket)
*   **`TelemetryFrameHeader`**
    *   **Размер / Выравнивание**: 16 байт / `align(8)`.
    *   **Поля**: `magic` ([u8; 4] = `"TELE"`), `tick` (u32), `count` (u32), `_padding` (u32).
    *   **Семантика**: Заголовок WebSocket фрейма телеметрии спайков.

#### Домен 5: IPC и Запекание (UDS IPC)
*   **`BakeRequest`**
    *   **Размер / Выравнивание**: 16 байт / `align(4)`.
    *   **Поля**: `magic` ([u8; 4] = `"BAKE"`), `zone_hash` (u32), `current_tick` (u32), `prune_threshold` (i16), `max_sprouts` (u16).
    *   **Семантика**: Триггер запекания для взаимодействия между нодой и `weaver-daemon`.

### §4.2. Трейты

#### `pub trait WireCast`
Предоставляет вспомогательные zero-cost методы для приведения POD типов:
*   `fn as_bytes(&self) -> &[u8]` — приведение структуры к байтовому срезу за $O(1)$.
*   `fn slice_as_bytes(slice: &[Self]) -> &[u8]` — приведение среза структур к байтовому срезу.
*   `fn from_bytes(buf: &[u8]) -> Result<&Self, WireError>` — приведение байтов к ссылке на структуру с проверкой выравнивания и длины.
*   `fn slice_from_bytes(buf: &[u8]) -> Result<&[Self], WireError>` — приведение байтов к срезу структур.

### §4.4. Константы и Магические Числа

Крейт не экспортирует публичных Rust констант. Магические числа пакетов вшиты в заголовки или проверяются на уровне протоколов:
- `b"TELE"` (Телеметрия)
- `b"GSIO"`, `b"GSOO"` (Ввод/вывод среды)
- `b"DOPA"` (Инъекции дофамина)
- `b"BAKE"` (Триггер запекания)

## §5. Доменная Логика

Крейт служит единым источником правды для сетевого и межпроцессного протоколов. Его выделение изолирует структуру сообщений от логики их передачи и кодогенерации SDK, предотвращая рассинхронизацию форматов между Rust и Python.

Фиксация Little-Endian формата и выравнивания под целевые архитектуры позволяет передавать и считывать данные без процессороемкой сериализации, копирования в куче или перестановки байтов.

## §6. Алгоритмы и Формулы

### §6.1. Трансляция AST в Python struct макеты (Кодогенерация)

**Вход**: Файлы исходного кода `src/*.rs` с AST-представлением Rust структур, объявленных через `decl_wire!`.
**Выход**: Текстовый файл Python модуля `contract_generated.py` со структурами `ctypes.LittleEndianStructure`.
**Детерминизм**: Да.

**Псевдокод алгоритма (build.rs):**
```rust
fn generate_python_contracts() {
    let files = list_rust_files("src/");
    let mut out = String::new();
    for file in files {
        let structs = parse_c_repr_structs(file);
        for s in structs {
            // Генерация ctypes структуры с жестким _pack_ (alignment)
            out.push_str(&format!(
                "class {}(ctypes.LittleEndianStructure):\n    _pack_ = {}\n    _fields_ = [\n",
                s.name, s.alignment
            ));
            for f in s.fields {
                out.push_str(&format!("        (\"{}\", {}),\n", f.name, translate_type(f.rust_type)));
            }
            out.push_str("    ]\n\n");
        }
    }
    write_file("contract_generated.py", out);
}
```

### §6.2. O(1) Zero-Copy Cast (Безопасное приведение)

**Вход**: Сырой буфер `payload: &[u8]`.
**Выход**: Типизированная ссылка `&TargetStruct` или срез `&[TargetStruct]`, либо ошибка `WireError`.

**Алгоритм:**
Аппаратное приведение типов (через `bytemuck`) без копирования требует двух проверок:
1. **Проверка длины**: `payload.len() % sizeof(TargetStruct) == 0` (или равенство для единичной структуры).
2. **Проверка выравнивания**: `payload.as_ptr() as usize % align_of(TargetStruct) == 0`.

## §7. Структуры Данных и Memory Layout

Все структуры выровнены и упакованы без скрытых компилятором паддингов.

### §7.1. SpikeBatchHeaderV2 Layout (16 байт, align 4)
| Offset | Size | Field | Type |
|---|---|---|---|
| 0x00 | 4 | `src_zone_hash` | `u32` |
| 0x04 | 4 | `dst_zone_hash` | `u32` |
| 0x08 | 4 | `epoch` | `u32` |
| 0x0C | 2 | `chunk_idx` | `u16` |
| 0x0E | 2 | `total_chunks` | `u16` |

| Offset | Size | Field | Type |
|---|---|---|---|

### §7.3. TelemetryFrameHeader Layout (16 байт, align 8)
| Offset | Size | Field | Type |

| 0x08 | 4 | `count` | `u32` |

## §8. Граничные Случаи и Особые Сценарии

### §8.1. Граничные значения

| # | Ситуация | Ожидаемое поведение |
|---|----------|-------------------|
| E-021 | **Несоответствие длины буфера**: Буфер слишком мал для приведения к структуре. | Метод `from_bytes` возвращает `WireError::BufferTooSmall`. |
| E-022 | **Невыровненный буфер (Alignment Mismatch)**: Адрес начала буфера в памяти не выровнен под целевой тип. | Возвращается `WireError::AlignmentMismatch`. Паника исключена. |
| E-023 | **Ошибки кодогенерации**: Недопустимый тип поля при сборке в `build.rs`. | `build.rs` вызывает панику при сборке, сообщая о неизвестном C-ABI типе. |

## §9. Ошибки

### §9.1. Перечисление ошибок

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum WireError {
    /// Неверный бинарный идентификатор (magic number)
    InvalidMagic { expected: [u8; 4], actual: [u8; 4] },
    /// Недостаточный размер буфера для кастинга структуры
    BufferTooSmall { expected: usize, actual: usize },
    /// Ошибка выравнивания памяти буфера
    AlignmentMismatch,
    /// Несовпадение версии формата
    VersionMismatch { expected: u32, actual: u32 },
    /// Ошибки семантической валидации
    ValidationError(&'static str),
}
```
