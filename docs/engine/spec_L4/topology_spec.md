## §1. Идентификация

| Поле | Значение |
|---|---|
| **Имя крейта** | `topology` |
| **Слой** | Слой 4 — Geometry, Growth & Connectome Generation (`L4`) |
| **Тип** | Library (`lib`) |
| **no_std** | Нет (`false`) — требуется динамическая память (`Vec`), сортировки и алгоритмы трехмерной геометрии |
| **Описание** | Крейт чистых алгоритмов пространственной геометрии и структурного роста графа. На этапе Stage A крейт реализует детерминированное размещение сом нейронов внутри одного шарда по правилам слоев, плотности и состава типов. На последующих этапах добавляются воксельные пространственные сетки, локальный рост аксонов (Stage B), диагностический dev-only вывод (Stage C) и планировщики биологических изменений Ночной Фазы (Stage N). |

## §1.1. Этапы реализации (Staged Scope)

Для обеспечения управляемой сложности и поэтапной верификации, функционал `topology` разделен на следующие этапы:

### Stage A — Single-Shard Soma Placement Core
**Фокус первой реализации**: Детерминированное размещение тел нейронов (сом) в границах одного шарда.
- Вход: `config::ShardConfig` и `types::MasterSeed`.
- Выход: Чистые Rust DTO со списком размещенных сом.
- Размещение с учетом `dimensions`, `layers`, `density`, `composition` и `neuron_types`.
- Инвариант: В одном вокселе шарда находится максимум одна сома.
- Изоляция: Выход в виде структур данных, без выполнения I/O операций на диск.
- RNG: Использование безстанционных хелперов и смесителей на базе `types::MasterSeed` без привлечения внешних RNG-крейтов.

### Stage B1 — Local Axon Growth Path Geometry
**Текущий проектируемый этап**: Построение геометрии путей аксонов внутри локального воксельного пространства одного шарда.
- Вход: `config::ShardConfig`, `SingleShardTopology` и `types::MasterSeed`.
- Выход: Чистые Rust DTO со списком сгенерированных 3D путей аксонов (`LocalGrowthResult`).
- Ограничение по длине пути: `layout::MAX_SEGMENTS_PER_AXON` (обычно 256, с учетом `segment_offset` от `1` до `255`).
- RNG: Детерминированное безстанционное PRNG хеширование для выбора направлений (Moore neighborhood).
- Препятствия: Сомы других нейронов и самопересечения.
- Границы: Выход за границы шарда останавливает рост с причиной `BoundaryReached`.

### Stage B2 — Local Synapse Formation
**Фокус второй реализации**: Построение in-memory DTO плана локальных синаптических связей.
- Вход: `config::ShardConfig`, `SingleShardTopology`, `LocalGrowthResult` и `types::MasterSeed`.
- Выход: Чистые Rust DTO плана связей (`LocalSynapsePlan`) без бинарной записи на диск.
- Дендритные входы: Не более `layout::MAX_DENDRITES` (128) live синапсов на каждую сому с детерминированным ранжированием и отсечением кандидатов при избытке.
- Межшардовые связи и ghost handover остаются в deferred-статусе.

### Stage C — Diagnostic Probe Artifacts
**Вспомогательный отладочный функционал**:
- Dev-only экспорт для визуальной и аналитической проверки плотности и качества распределения сом.
- Минимальный формат файла отчета: `artifacts/topology_probe_somas.csv`
- Обязательный заголовок и состав колонок: `soma_id,layer_index,variant_id,x,y,z`
- Не является частью основного продуктового API и не требует интеграции с AxiCAD/редакторами.

### Stage N — Night Phase Pure Planning
**Алгоритмы планирования Ночной Фазы**:
- Разработка чистых хостовых планировщиков для Ночной Фазы (прунинг, ранжирование спраутов, уплотнение SoA-плоскостей, ghost handover).
- Строгая чистота: 0 прямых записей в SHM, 0 зависимостей от `compute-api`, работа строго с неизменяемыми видами `NightWorkingViewRef`.

## §4. Система Координат и Пространственная Вокселизация (Coordinate Policy)

1. **Единое Воксельное Пространство (Voxel Space)**: Все координаты внутри шарда хранятся в виде трехмерных дискретных вокселей в структуре `PackedPosition`:
   - Координата $X$: диапазон `0..1023` (10 бит).
   - Координата $Y$: диапазон `0..1023` (10 бит).
   - Координата $Z$: диапазон `0..255` (8 бит).
   - Тип нейрона / маркер: диапазон `0..15` (4 бита).
2. **Создание Позиций**: Все структуры позиций сом создаются исключительно через API `types::PackedPosition`. Прямое ручное побитовое дублирование или сдвиги вне API `types` строго запрещены.
3. **Разрешенная Геометрия**: Для Stage A все координаты размещения сом вычисляются детерминированно в пределах дискретной сетки шарда `[0..w) x [0..d) x [0..h)`.

## §5. Алгоритмы Размещения Сом (Stage A Core Algorithms)

### §5.1. Алгоритм Нарезки Слоев (Layer Slicing)
Для точного распределения воксельного пространства Z-оси шарда по анатомическим слоям используется кумулятивный алгоритм:
1. Вычисляется высота шарда `shard_h = dimensions.h`.
2. Слои обрабатываются в порядке их объявления. Накапливается кумулятивная доля высоты:
   - `cumulative_before` — сумма долей всех предыдущих слоев.
   - `cumulative_after = cumulative_before + current_layer.height_pct`.
3. Границы слоя на оси Z рассчитываются как:
   - `z_start = floor(cumulative_before * shard_h)` (включая).
   - `z_end = floor(cumulative_after * shard_h)` (исключая).
4. Последний объявленный слой принудительно заканчивается на `shard_h` (`z_end = shard_h`).
5. Диапазон высоты слоя полуоткрытый: `[z_start, z_end)`. Высота слоя в вокселях составляет `layer_h_voxels = z_end - z_start`.
6. **Защитная проверка**: Если в результате округления `layer_h_voxels == 0` при ненулевой плотности `density > 0.0`, генерация прерывается с ошибкой `TopologyError::LayerGeometryError`.

### §5.2. Расчет Количества Сом (Soma Count Derivation)
Все расчеты емкостей (`layer_capacity`), количеств нейронов (`layer_soma_count`) и плоских смещений внутри крейта выполняются с использованием безопасной checked-арифметики (`usize`/`u64`). При возникновении переполнения (например, при перемножении сверхбольших размеров мира) вычисление немедленно прерывается и возвращается ошибка `TopologyError::CapacityOverflow`.

Для каждого слоя количество размещаемых сом рассчитывается по правилам:
1. Вычисляется воксельная емкость слоя:
   `layer_capacity = dimensions.w * dimensions.d * layer_h_voxels` (через checked-умножение).
2. Целевое количество сом в слое рассчитывается как:
   `layer_soma_count = floor(layer_capacity as f64 * density as f64) as usize`
   При этом плотность `density` гарантированно является конечным числом (`is_finite()`) в диапазоне `0.0..=1.0` благодаря предварительной валидации на уровне крейта `config`.
3. Распределение по типам нейронов в слое выполняется по алгоритму **Largest-Remainder (Квота Гамильтона)**:
   - Шаг 1: Для каждого типа нейрона в композиции слоя рассчитывается идеальная доля: `ideal = layer_soma_count * composition.share`.
   - Шаг 2: Каждому типу выделяется гарантированное целое число сом: `allocated[type_name] = floor(ideal)`. Накапливается сумма выделенных тел.
   - Шаг 3: Оставшиеся сомы (`remainder = layer_soma_count - sum(allocated)`) распределяются по 1 штуке среди типов нейронов в порядке убывания дробной части `ideal - floor(ideal)`.
   - Шаг 4 (Tie-breaking): При равных дробных частях приоритет отдается типу с меньшим `variant_id` (индексом в массиве `neuron_types` конфигурации шарда).
4. **Инвариант**: Сумма распределенных количеств по всем типам нейронов в слое обязана строго равняться `layer_soma_count`. Превышение емкости слоя `layer_soma_count > layer_capacity` аппаратно невозможно благодаря валидации `density <= 1.0` in `config`.

### §5.3. Стабильный Порядок Назначения Soma ID
Для обеспечения воспроизводимости отладки все сомы нумеруются строго последовательно (`soma_id` от `0` до `total_somas - 1`) в следующем лексикографическом порядке сортировки:
1. Индекс анатомического слоя (`layer_index`);
2. Идентификатор типа нейрона (`variant_id`), который строго равен индексу `NeuronType` в массиве `config::ShardConfig.neuron_types`;
3. Локальный порядковый номер сом внутри данного типа в текущем слое (`local_ordinal`).

### §5.4. Детерминированное Размещение (Deterministic Soma Placement)
Для каждой сомы с уникальным набором `(layer_index, variant_id, local_ordinal)` генерируется координата вокселя по следующему алгоритму:
1. Вычисляется плоская емкость слоя:
   `layer_capacity = shard_w * shard_d * layer_h_voxels` (где `shard_w = dimensions.w`, `shard_d = dimensions.d`).
2. Вычисляется детерминированное стартовое смещение с помощью смесителя хеша:
   `start = deterministic_hash(seed, layer_index, variant_id, local_ordinal) % layer_capacity` (выполняется с помощью побитового смешивания `hash` / `mix` примитивов из `types`).
3. Выполняется детерминированное зондирование по всей емкости слоя (для `attempt` в диапазоне `0..layer_capacity`):
   - Вычисляется плоский индекс вокселя:
     `flat_idx = (start + attempt) % layer_capacity`
   - Индекс преобразуется в трехмерные локальные координаты вокселя:
     `z = z_start + flat_idx / (shard_w * shard_d)`
     `rem = flat_idx % (shard_w * shard_d)`
     `y = rem / shard_w`
     `x = rem % shard_w`
   - Если воксель с координатами `(x, y, z)` свободен, сома размещается в нем, и генерация для данной сомы успешно завершается.
4. Если после перебора всех `layer_capacity` попыток свободный воксель не найден, возвращается ошибка `TopologyError::LayerCapacityExceeded`.
Silent drop (молчаливый пропуск размещения сомы) строго запрещен.

### §5.5. Алгоритмы Роста Аксонов (Stage B1 Local Axon Growth)
Алгоритм детерминированно строит траекторию роста аксона для каждой сомы независимо:
1. **Инициализация**:
   - Для сомы `soma` извлекается ее тип:
     Если `soma.variant_id as usize >= config.neuron_types.len()`, генерация аварийно завершается с ошибкой `TopologyError::UnknownNeuronType { variant_id: soma.variant_id }`.
     Иначе `source_type = &config.neuron_types[soma.variant_id as usize]`.
   - Стартовая позиция аксона: воксель сомы `soma.position`.
   - Воксель сомы считается занятым origin-точкой пути (не может повторно посещаться).
   - Номер текущего шага `step_index` инициализируется равным `1`.
2. **Moore Neighborhood (26 кандидатов) и их упорядочивание**:
   На каждом шаге оцениваются 26 потенциальных направлений движения $u_i = (dx_i, dy_i, dz_i)$, где $dx_i, dy_i, dz_i \in \{-1, 0, 1\}$, исключая $(0, 0, 0)$.
3. **Целочисленный расчет Score (Steering Model)**:
   Для каждого из 26 кандидатов вычисляется целочисленный балл `score_q` (в типе `i64`):
   $$score\_q = inertia\_q \cdot \text{dot}(d_{prev}, u_i) + vertical\_q \cdot dz_i + \frac{jitter\_q \cdot jitter\_unit}{65535}$$
4. **Сортировка приоритетов кандидатов**:
   Все 26 кандидатов сортируются по значению `score_q` в порядке убывания, а при равенстве — по `candidate_index` в порядке возрастания.
5. **Выбор направления и правила препятствий / границ (Stop Reasons)**:
   - Вне геометрических границ -> `AxonGrowthStopReason::BoundaryReached`.
   - Препятствие (другая сома) -> Пропуск.
   - Самопересечение -> Пропуск.
6. **Условия остановки роста**:
   - **MaxLengthReached**: достигнуто 255 сегментов.
   - **BoundaryReached**: выход за границы шарда.
   - **Blocked**: все направления заблокированы.

### §5.6. Локальное формирование синапсов (Stage B2)
Задача `topology` — детерминированно сформировать список кандидатов в синаптические контакты и отобрать из них не более 128 живых связей для каждого целевого нейрона.
1. **Поиск кандидатов**: сегмент аксона находится в пределах `dendrite_radius_um` (сконвертированного в воксели с округлением вверх) от целевой сомы.
2. **Сортировка**: по возрастанию расстояния в квадрате, затем по возрастанию смещения сегмента аксона, затем по возрастанию ID сомы.
3. **Назначение весов**:
   - Возбуждающие синапсы положительные, тормозные — отрицательные.
   - Входной `initial_synapse_weight` переводится из Charge-scale в Mass Domain (`i32`) сдвигом влево на 16 бит. Нулевой вес заменяется на 1.

### §5.7. Алгоритмы планирования Ночной Фазы (Stage N Night Pure Planning)

Алгоритмы Stage N являются чисто вычислительными функциями планирования синаптических и топологических изменений, выполняющимися на хосте в рамках Ночной Фазы. Они не имеют побочных эффектов (не производят прямую запись в SHM, не используют mmap или compute-бэкенды) и работают исключительно с неизменяемыми срезами/представлениями `NightWorkingViewRef`.

#### §5.7.1. Формула скоринга кандидатов спраутинга (Scoring Formula)
При поиске потенциальных синапсов (спраутинга) каждый кандидат оценивается по целочисленной формуле скоринга в формате с фиксированной запятой (FP 16.16):
$$score\_fixed = w\_explore \cdot jitter\_unit + w\_distance \cdot dist\_sq + w\_power \cdot power\_fixed$$

Где:
- $w\_explore, w\_distance, w\_power$ — целочисленные коэффициенты FP 16.16 из параметров `SproutWeightParams`.
- $dist\_sq$ — квадрат расстояния в вокселях между сегментом аксона и целевой сомой.
- $jitter\_unit$ — 16-битный псевдослучайный шум `0..=65535`, сгенерированный детерминированно на основе `MasterSeed`, текущей эпохи, ID шарда и ID целевой сомы.
- $power\_fixed$ — средняя синаптическая масса дендритных входов целевого нейрона (целое число u16, ограниченное сверху 65535):
$$power\_fixed = \text{min}\left(65535, \frac{\sum_{j=0}^{127} |weight_j|}{128}\right)$$

Все промежуточные операции вычисляются в типе `i64` с использованием checked-арифметики для предотвращения переполнений. Сортировка кандидатов производится по значению `score_fixed` в порядке убывания. При равенстве скора приоритет разрешается с использованием `SproutRankKey`.

#### §5.7.2. Ранжирование спраутов (`SproutRankKey` / `cmp_rank`)
Для детерминированного tie-break ранжирования кандидатов используется вспомогательный ключ `SproutRankKey`. **Сортировка по умолчанию через derive `Ord` для него категорически запрещена**. Вместо этого реализуется явная функция сравнения `cmp_rank`:

```rust
pub struct SproutRankKey {
    pub score_fixed: i64,
    pub power_fixed: u16,
    pub target_soma_id: u32,
    pub dendrite_slot: u8,
}

pub fn cmp_rank(a: &SproutRankKey, b: &SproutRankKey) -> core::cmp::Ordering {
    // 1. По убыванию score_fixed (score DESC)
    let ord = b.score_fixed.cmp(&a.score_fixed);
    if ord != core::cmp::Ordering::Equal {
        return ord;
    }
    // 2. По убыванию power_fixed (power DESC)
    let ord = b.power_fixed.cmp(&a.power_fixed);
    if ord != core::cmp::Ordering::Equal {
        return ord;
    }
    // 3. По возрастанию target_soma_id (soma ASC)
    let ord = a.target_soma_id.cmp(&b.target_soma_id);
    if ord != core::cmp::Ordering::Equal {
        return ord;
    }
    // 4. По возрастанию dendrite_slot (slot ASC)
    a.dendrite_slot.cmp(&b.dendrite_slot)
}
```


#### §5.7.3. Планирование прунинга (Pruning Plan)
Чистая функция `plan_pruning` сканирует срез синаптических весов шарда и определяет индексы элементов, вес которых по абсолютной величине упал ниже порога `prune_threshold`:

```rust
pub fn plan_pruning(
    weights: &[i32],
    prune_threshold: u32,
) -> Vec<usize> {
    let mut prune_indices = Vec::new();
    for (idx, &w) in weights.iter().enumerate() {
        if w != 0 && (w.unsigned_abs() < prune_threshold) {
            prune_indices.push(idx);
        }
    }
    prune_indices
}
```

#### §5.7.4. Выбор свободного дендритного слота (`choose_dendrite_slot`)
Выбор осуществляется на основе сканирования списка таргетов целевого нейрона. Функция находит первый неактивный слот (сырой нуль `0` или tombstone `EMPTY_PIXEL`):

```rust
pub fn choose_dendrite_slot(targets: &[PackedTarget]) -> Option<u8> {
    for (idx, target) in targets.iter().take(128).enumerate() {
        if target.is_inactive() {
            return Some(idx as u8);
        }
    }
    None
}
```

#### §5.7.5. Планирование уплотнения (Compaction Plan)
При наличии мертвых связей (tombstones `EMPTY_PIXEL`) внутри дендритного ряда нейрона, чистая функция строит план сдвига активных элементов влево, вытесняя неактивные слоты в конец ряда.

```rust
pub struct CompactionPlan {
    /// Пары индексов (from, to) для перемещения активных синапсов
    pub moves: Vec<(u8, u8)>,
    /// Количество очищаемых слотов в хвосте ряда
    pub tail_clear_count: u8,
}
```
План строится независимо для каждого нейрона: все активные элементы (где `!target.is_inactive()`) сдвигаются на минимально возможные индексы, сохраняя исходный относительный порядок следования.

#### §5.7.6. Черновик передачи Ghost-связей (`GhostHandoverDraft`)
Для межшардовых связей, выходящих за границы текущего шарда, планировщик формирует нейтральный черновик переходов:

```rust
pub struct GhostHandoverDraft {
    pub source_soma_id: u32,
    pub target_shard_id: u32,
    pub target_soma_id: u32,
    pub segment_offset: u8,
}
```
Данный черновик служит выходом алгоритма планирования и передается `weaver-daemon` для последующей материализации.

## §8. Требуемые Инварианты (Stage A, Stage B1, Stage B2 & Stage N)

- **INV-TOPO-001**: *Ограничение плотности вокселей*. В одном вокселе трехмерной сетки шарда может находиться не более одной сомы нейрона.
- **INV-TOPO-002**: *Контроль емкости слоев*. Если физическое место для размещения требуемого количества сом исчерпано, возвращается `TopologyError::LayerCapacityExceeded`.
- **INV-TOPO-003**: *Соответствие типов*. Поле `variant_id` in `PlacedSoma` строго соответствует индексу `NeuronType` в массиве `ShardConfig.neuron_types`.
- **INV-TOPO-004**: *Детерминизм вывода*. Один и тот же набор `SingleShardTopologyInput` обязан возвращать побитово идентичный результат `SingleShardTopology` на любых поддерживаемых архитектурах.
- **INV-TOPO-B1-001**: *Детерминизм траектории*. При одинаковых входных данных путь роста аксона полностью воспроизводим.
- **INV-TOPO-B1-002**: *Лимит длины*. Количество сегментов пути аксона не превышает 255 сегментов.
- **INV-TOPO-B2-001**: *Лимит входов дендритов*. Каждая целевая сома содержит не более 128 живых (активных) синапсов.
- **INV-TOPO-B2-002**: *Детерминированное ранжирование*. Выбор топ-128 синапсов при избытке кандидатов осуществляется на основе строго упорядоченного ключа (расстояние, offset, ID).
- **INV-TOPO-B2-003**: *Запрет самосинапсов*. Ни один синапс не может связывать сому саму с собой.
- **INV-TOPO-N-001**: *Чистота выполнения*. Алгоритмы Stage N не производят никаких записей в разделяемую память хоста (SHM) или VRAM.
- **INV-TOPO-N-002**: *Запрет derive Ord*. Сортировочные структуры ранжирования спраутов `SproutRankKey` не используют автоматический derive Ord/PartialOrd.
- **INV-TOPO-N-003**: *Детерминированный скоринг*. При одинаковом `MasterSeed` и эпохе псевдослучайный шум `jitter_unit` и формулы скоринга спраутов возвращают строго воспроизводимый результат.
- **INV-TOPO-N-004**: *Сохранение порядка сдвига*. Алгоритм уплотнения (Compaction) сдвигает активные синапсы строго влево, сохраняя их относительный порядок.

## §9. Golden Tests / Обязательная Матрица Тестирования (Stage A, Stage B1, Stage B2 & Stage N)

### Тесты этапа Stage A
1. **Воспроизводимость размещения сом (`test_topology_reproducible_soma_placement`)**: Верификация 100% совпадения координат сом при повторных запусках с одинаковым `MasterSeed`.
2. **Детерминизм изменения семени (`test_topology_seed_changes_placement_but_preserves_counts`)**: Проверка, что другое семя `MasterSeed` меняет позиции сом, но оставляет неизменным их число по типам.
3. **Отсутствие коллизий вокселей (`test_topology_no_voxel_collisions`)**: Проверка инварианта `INV-TOPO-001` — не более одной сомы на воксель.
4. **Соблюдение плотности слоев (`test_topology_layer_density_respected`)**: Проверка соответствия числа размещенных нейронов расчетной плотности.
5. **Детерминизм Largest-Remainder (`test_topology_composition_largest_remainder_deterministic`)**: Верификация математики Гамильтона распределения остатка.

### Тесты этапа Stage B1
6. **Целочисленный расчет направления (`test_topology_fixed_point_steering_no_float_runtime`)**: Выбор направления Moore neighborhood дает точный fixed-point результат без float.
7. **Воспроизводимость путей роста (`test_topology_reproducible_growth_path`)**: Верификация 100% идентичности сегментов аксона при повторной трассировке.

### Тесты этапа Stage B2
8. **Детерминизм формирования синапсов (`test_topology_formation_deterministic_same_seed`)**: Проверка полной идентичности сгенерированного плана связей при повторных вызовах.
9. **Лимит MAX_DENDRITES и ранжирование (`test_topology_formation_max_dendrites_cap`)**: Проверка, что для одного нейрона формируется не более 128 связей, а при их избытке отбор выполняется строго по сортировочному ключу.

### Тесты этапа Stage N
10. **Детерминированное сравнение рангов спраутов (`test_sprout_ranking_logic`)**: Проверка корректности кастомного компаратора `cmp_rank` для `SproutRankKey` (score DESC, power DESC, target_soma_id ASC, dendrite_slot ASC), подтверждающая отсутствие derive Ord.
11. **Вычисление средней массы дендритов FP 16.16 (`test_compute_power_fixed`)**: Проверка математической формулы средней синаптической массы дендритного дерева `min(65535, sum_abs(weights) / 128)`.
12. **Скоринг спраутов с фиксированной точкой (`test_sprout_scoring_fixed_point`)**: Проверка формулы `score_fixed` для различных комбинаций весов, checked-арифметика в i64.
13. **Построение плана уплотнения (`test_compaction_plan_construction`)**: Проверка алгоритма сдвига активных элементов с вытеснением tombstones.
