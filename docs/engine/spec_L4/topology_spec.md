## §3. Инварианты

### §3.1. Структурные инварианты

- **INV-TOPO-001**: *Ограничение плотности вокселей*.
  - *Обоснование*: В одном вокселе трехмерной сетки может находиться максимум одна сома нейрона. Это позволяет использовать координаты вокселя в качестве O(1) уникального идентификатора для быстрого пространственного поиска и хэширования без коллизий.
  - *Следствие нарушения*: Коллизии адресации, непредсказуемое поведение при маппинге `DenseIndex` -> `PackedPosition`.
  - *Где проверяется*: Исключается алгоритмом стохастического размещения `place_somas` (создание и перемешивание воксельного пула).

### §3.2. Семантические инварианты

- **INV-TOPO-004**: *Sprouting Density Invariant (GPU Visibility)*.
  - *Обоснование*: При поиске свободного дендритного слота для новой связи цикл на CPU обязан последовательно сканировать массив от 0 до 127 и записывать новый синапс в первый встреченный слот, где `target == 0` или `target == EMPTY_PIXEL`. GPU-ядро Day Phase использует оптимизацию Early Exit. Запись после пустого слота сделает синапс аппаратно невидимым.
  - *Следствие нарушения*: Silent Data Loss — новообразованные синапсы не считываются GPU в горячем цикле.
  - *Где проверяется*: runtime check в функции `sprout_connections` (вызов `find_first_empty_slot`).

- **INV-TOPO-005**: *Dead on Arrival (DoA) Protection*.
  - *Обоснование*: При рождении синапса его начальный вес обязан быть сдвинут в Mass Domain (`initial_weight << 16`). Если сдвинутый вес меньше или равен порогу прунинга `prune_threshold << 16`, синапсу принудительно выдается спасательный капитал.
  - *Следствие нарушения*: 100% смертность новых синапсов при ночной консолидации.
  - *Где проверяется*: runtime assert в функции `sprout_connections`.

- **INV-TOPO-006**: *Закон Дейла при Sprouting*.
  - *Обоснование*: Знак начального веса синапса определяется исключительно типом нейрона-владельца аксона (возбуждающий (+) или тормозный (-)), а не принимающей сомы.
  - *Следствие нарушения*: Дестабилизация баланса возбуждения/торможения.
  - *Где проверяется*: runtime assert при создании связи в `sprout_connections`.

- **INV-TOPO-007**: *Единственность связи сома-аксон (Rule of Uniqueness)*.
  - *Обоснование*: Сома может иметь максимум одну связь с конкретным `Axon_ID`. Дублирование сигналов бессмысленно, дефицитные слоты (128) нужно тратить на разные источники.
  - *Следствие нарушения*: Быстрое исчерпание дендритных слотов дублирующими сигналами.
  - *Где проверяется*: runtime check при поиске дубликатов в `sprout_connections` (вызов `is_duplicate_axon`).

- **INV-TOPO-009**: *Activity-Based Nudging (Axon growth gate)*.
  - *Обоснование*: Локальный аксон может сделать шаг роста во время Ночной Фазы только в том случае, если его сома-владелец спайковала в течение предшествующей Дневной Фазы (`soma_flags & 0x01 == 1`).
  - *Следствие нарушения*: Избыточный расход CPU-времени в Ночную Фазу.
  - *Где проверяется*: runtime check в алгоритме `nudge_living_axons`.

- **INV-TOPO-010**: *Inertial Nudging для Ghost-аксонов*.
  - *Обоснование*: Ghost-аксоны не имеют локальной сомы (`soma_idx == usize::MAX`) и обязаны расти каждую ночь безусловно по вектору своей инерции, пока счетчик `remaining_steps` не достигнет нуля.
  - *Следствие нарушения*: Остановка роста межшардовых связей.
  - *Где проверяется*: runtime check в алгоритме `nudge_living_axons`.

- **INV-BAKER-003**: *Детерминизм индекса маршрутизации*.
  - *Обоснование*: Для обеспечения bit-to-bit воспроизводимости `.axic` архива макро-маршрутизатор обязан строить индекс позиций на базе `BTreeMap`, а не `HashMap`.
  - *Следствие нарушения*: Недетерминированный порядок выходных сетевых связей.
  - *Где проверяется*: Вызов `build_soma_index` в `routing.rs`.

### §4.3. Функции

#### `pub fn place_somas(bounds: (u32, u32, u32), layers: &[config::LayerConfig], rng: &mut rand_chacha::ChaCha8Rng) -> Result<Vec<types::PackedPosition>, crate::error::TopologyError>`
- **Назначение**: Стохастическое бесколлизионное размещение сом в воксельной сетке на основе параметров слоев анатомии.
- **Сложность**: O(N) в среднем, где N — бюджет сом.
- **Паника**: Никогда (при переполнении плотности возвращает `TopologyError::PlacementCollision`).

#### `pub fn cone_tracing(soma_pos: types::PackedPosition, target_z: u32, weights: &SteeringWeights, segment_length_voxels: f32, rng: &mut ChaCha8Rng) -> Vec<types::PackedPosition>`
- **Назначение**: Прокладывает направленный аксонный путь (Trunk Phase) от сомы к целевому Z-слою.
- **Сложность**: O(S), где S — количество сегментов до цели (максимум `MAX_SEGMENTS_PER_AXON = 256`).
- **Паника**: Никогда.

#### `pub fn nudge_living_axons(living: &mut [LivingAxon], soma_flags: &[u8], bounds: (u32, u32, u32), segment_length_voxels: f32) -> Vec<GrowthEvent>`
- **Назначение**: Сдвигает кончики активных локальных аксонов и Ghost-аксонов на 1 сегмент.
- **Сложность**: O(A), где A — количество аксонов.
- **Паника**: Никогда.

#### `pub fn sprout_connections(active_somas: &[usize], existing_targets: &[u32], padded_n: usize, grid: &SpatialGrid, neuron_types: &[config::NeuronType], prune_threshold: i16, soma_positions: &[types::PackedPosition]) -> Vec<NewSynapse>`
- **Назначение**: Поиск контактов (En Passant) в воксельной сетке для активных сом.
- **Сложность**: O(N_active * K), где K — количество сегментов в радиусе поиска.
- **Паника**: Никогда.

#### `pub fn route_ghost_atlas(source_gxo_somas: &[u32], width: u32, height: u32, target_bounds: (u32, u32, u32), entry_z: EntryZ, target_type_idx: u8, soma_positions: &[PackedPosition]) -> Result<Vec<GhostConnection>, TopologyError>`
- **Назначение**: Проецирует 2D пиксели вывода источника на сомы получателя в целевом Z-столбце шарда (AOT Atlas Routing).
- **Сложность**: O(W * H * Z), где WxH — размеры матрицы, Z — глубина сканирования столбца.
- **Паника**: Никогда.

### §4.4. Константы

Константы вшиты в алгоритмы или импортируются из `layout` / `types`:
- `MAX_DENDRITES` (128): Физический предел синапсов на одну сому.
- `MAX_SEGMENTS_PER_AXON` (256): Аппаратный предел шагов роста аксона.
- `EMPTY_PIXEL` (`0xFFFF_FFFF`): Маркер пустого пикселя для GPU.

## §5. Доменная Логика

Крейт `topology` — это движок трехмерной геометрии и структурной пластичности графа. Он транслирует биологические 3D-законы роста и поиска соседей в плоские, жестко выровненные одномерные индексы (SoA) для симулятора.

Выделение тяжелой 3D-математики (воксельные сетки, трассировка) в изолированный крейт решает архитектурный конфликт: видеокарта GPU эффективно работает только с 1D-векторами, тогда как нейросеть развивается в объеме. Крейт переносит всю ресурсоемкую работу с пространством на фазу AOT-компиляции (Baking) и Ночную Фазу (Night Phase).

## §6. Алгоритмы и Формулы

### §6.1. Алгоритм стохастического размещения (Stochastic Placement)

Вместо медленного классического reject-sampling с риском зависаний, алгоритм генерирует пул вокселей слоя, перемешивает его (Shuffle) и последовательно выдает координаты. Это дает аппаратную гарантию отсутствия коллизий (INV-TOPO-001) и точного выполнения квот.

### §6.2. Конусная трассировка (Cone Tracing)

На каждом шаге вычисляется направляющий вектор:
```math
\text{v\_steer} = \text{normalize}(\text{v\_global} \cdot \text{weights.global} + \text{v\_noise} \cdot \text{weights.noise})
```
Здесь `v_noise` — равномерно распределенный вектор единичной сферы.

### §6.3. Инерционный сдвиг и Activity Gate

Локальные аксоны сдвигаются только если их сома спайковала в Дневной Фазе (активность 0-го бита флагов сомы). Ghost-аксоны сдвигаются безусловно. При выходе за границы шарда генерируется `GhostPacket`.

### §6.4. Структурная пластичность (Sprouting)

Поиск синапсов выполняется через `SpatialGrid.find_in_radius`.
Для найденного кандидата:
1. Выполняется проверка на уникальность (INV-TOPO-007).
2. Запись производится в первый свободный SoA слот (INV-TOPO-004).
3. Начальный вес сдвигается: `weight = initial_weight << 16` и страхуется от прунинга (INV-TOPO-005).
4. Накладывается Dale's Law знак на основе типа аксона (INV-TOPO-006).

## §8. Граничные Случаи и Особые Сценарии

### §8.1. Граничные значения

| # | Ситуация | Ожидаемое поведение |
|---|---|---|
| E-077 | Экстремальная плотность вокселей | `place_somas` возвращает ошибку `TopologyError::PlacementCollision`. |
| E-078 | Длина аксона при росте превышает 256 сегментов | Цикл `cone_tracing` ограничен `MAX_SEGMENTS_PER_AXON`. По достижении лимита генерация останавливается, предотвращая переполнение. |
| E-079 | Отсутствие подходящих сом в столбце | `route_ghost_atlas` записывает маркер `EMPTY_PIXEL` (`0xFFFF_FFFF`) для организации GPU Early Exit. |
| E-080 | Выход аксона за физические границы шарда | `nudge_living_axons` останавливает локальный рост и возвращает `GrowthEvent::OutOfBounds(GhostPacket)`. |
| E-081 | Количество связей на сому превышает 128 | `find_first_empty_slot` возвращает `None`, спраутинг для данного нейрона останавливается. |
| E-082 | Стартовый вес синапса ниже порога прунинга | Срабатывает DoA защита: вес форсированно устанавливается в `prune_threshold + 100`. |
