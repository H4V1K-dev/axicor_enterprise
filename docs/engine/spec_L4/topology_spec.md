## §1. Идентификация

| Поле | Значение |
|---|---|
| **Имя крейта** | `topology` |
| **Слой** | Слой 4 — Geometry, Growth & Connectome Generation (`L4`) |
| **Тип** | Library (`lib`) |
| **no_std** | Нет (`false`) — требуется динамическая память (`Vec`), сортировки и алгоритмы трехмерной геометрии |
| **Описание** | Крейт чистых алгоритмов пространственной геометрии и структурного роста графа. На этапе Stage A крейт реализует детерминированное размещение сом нейронов внутри одного шарда по правилам слоев, плотности и состава типов. На последующих этапах добавляются воксельные пространственные сетки, локальный рост аксонов (Stage B) и диагностический dev-only вывод (Stage C). |

## §1.1. Этапы реализации (Staged Scope)

Для обеспечения управляемой сложности и поэтапной верификации, функционал `topology` разделен на следующие этапы:

### Stage A — Single-Shard Soma Placement Core
**Фокус первой реализации**: Детерминированное размещение тел нейронов (сом) в границах одного шарда.
- Вход: `config::ShardConfig` и `types::MasterSeed`.
- Выход: Чистые Rust DTO со списком размещенных сом.
- Размещение с учетом `dimensions`, `layers`, `density`, `composition` и `neuron_types`.
- Инвариант: В одном вокселе шарда находится максимум одна сома.
- Изоляция: Выход в виде структур данных, без выполнения I/O операций на диск.
- RNG: Использование безстанционных хелперов и смесителей на базе `types::MasterSeed` без привлечения внешних RNG-крейтов.

### Stage B1 — Local Axon Growth Path Geometry
**Текущий проектируемый этап**: Построение геометрии путей аксонов внутри локального воксельного пространства одного шарда.
- Вход: `config::ShardConfig`, `SingleShardTopology` и `types::MasterSeed`.
- Выход: Чистые Rust DTO со списком сгенерированных 3D путей аксонов (`LocalGrowthResult`).
- Ограничение по длине пути: `layout::MAX_SEGMENTS_PER_AXON` (обычно 256, с учетом `segment_offset` от `1` до `255`).
- RNG: Детерминированное безстанционное PRNG хеширование для выбора направлений (Moore neighborhood).
- Препятствия: Сомы других нейронов и самопересечения.
- Границы: Выход за границы шарда останавливает рост с причиной `BoundaryReached`.

### Stage B2 — Synapse Materialization & Pruning (Deferred)
**Последующий этап**:
- Построение пространственных индексов дендритов и en-passant синапсов.
- Запись в SoA-структуры памяти (`PackedTarget` и т.д.).
- Локальный расчет спраутинга и прунинга связей.

### Stage C — Diagnostic Probe Artifacts
**Вспомогательный отладочный функционал**:
- Dev-only экспорт для визуальной и аналитической проверки плотности и качества распределения сом.
- Минимальный формат файла отчета: `artifacts/topology_probe_somas.csv`
- Обязательный заголовок и состав колонок: `soma_id,layer_index,variant_id,x,y,z`
- Не является частью основного продуктового API и не требует интеграции с AxiCAD/редакторами.

### Out of Scope для первой реализации (Stage A):
- Межшардовое размещение (cross-shard placement);
- Маршрутизация межшардовых трактов (tract routing);
- Передача аксонов на границах (ghost handover);
- Физическое изменение SoA-массивов памяти в VRAM/RAM (монополия `weaver-daemon` / `compute`);
- Упаковка `.axic` архивов и VFS-операции;
- Сетевое, IPC или Wire взаимодействие.

## §4. Система Координат и Пространственная Вокселизация (Coordinate Policy)

1. **Единое Воксельное Пространство (Voxel Space)**: Все координаты внутри шарда хранятся в виде трехмерных дискретных вокселей в структуре `PackedPosition`:
   - Координата $X$: диапазон `0..1023` (10 бит).
   - Координата $Y$: диапазон `0..1023` (10 бит).
   - Координата $Z$: диапазон `0..255` (8 бит).
   - Тип нейрона / маркер: диапазон `0..15` (4 бита).
2. **Создание Позиций**: Все структуры позиций сом создаются исключительно через API `types::PackedPosition`. Прямое ручное побитовое дублирование или сдвиги вне API `types` строго запрещены.
3. **Разрешенная Геометрия**: Для Stage A все координаты размещения сом вычисляются детерминированно в пределах дискретной сетки шарда `[0..w) x [0..d) x [0..h)`.

## §5. Алгоритмы Размещения Сом (Stage A Core Algorithms)

### §5.1. Алгоритм Нарезки Слоев (Layer Slicing)
Для точного распределения воксельного пространства Z-оси шарда по анатомическим слоям используется кумулятивный алгоритм:
1. Вычисляется высота шарда `shard_h = dimensions.h`.
2. Слои обрабатываются в порядке их объявления. Накапливается кумулятивная доля высоты:
   - `cumulative_before` — сумма долей всех предыдущих слоев.
   - `cumulative_after = cumulative_before + current_layer.height_pct`.
3. Границы слоя на оси Z рассчитываются как:
   - `z_start = floor(cumulative_before * shard_h)` (включая).
   - `z_end = floor(cumulative_after * shard_h)` (исключая).
4. Последний объявленный слой принудительно заканчивается на `shard_h` (`z_end = shard_h`).
5. Диапазон высоты слоя полуоткрытый: `[z_start, z_end)`. Высота слоя в вокселях составляет `layer_h_voxels = z_end - z_start`.
6. **Защитная проверка**: Если в результате округления `layer_h_voxels == 0` при ненулевой плотности `density > 0.0`, генерация прерывается с ошибкой `TopologyError::LayerGeometryError`.

### §5.2. Расчет Количества Сом (Soma Count Derivation)
Все расчеты емкостей (`layer_capacity`), количеств нейронов (`layer_soma_count`) и плоских смещений внутри крейта выполняются с использованием безопасной checked-арифметики (`usize`/`u64`). При возникновении переполнения (например, при перемножении сверхбольших размеров мира) вычисление немедленно прерывается и возвращается ошибка `TopologyError::CapacityOverflow`.

Для каждого слоя количество размещаемых сом рассчитывается по правилам:
1. Вычисляется воксельная емкость слоя:
   `layer_capacity = dimensions.w * dimensions.d * layer_h_voxels` (через checked-умножение).
2. Целевое количество сом в слое рассчитывается как:
   `layer_soma_count = floor(layer_capacity as f64 * density as f64) as usize`
   При этом плотность `density` гарантированно является конечным числом (`is_finite()`) в диапазоне `0.0..=1.0` благодаря предварительной валидации на уровне крейта `config`.
3. Распределение по типам нейронов в слое выполняется по алгоритму **Largest-Remainder (Квота Гамильтона)**:
   - Шаг 1: Для каждого типа нейрона в композиции слоя рассчитывается идеальная доля: `ideal = layer_soma_count * composition.share`.
   - Шаг 2: Каждому типу выделяется гарантированное целое число сом: `allocated[type_name] = floor(ideal)`. Накапливается сумма выделенных тел.
   - Шаг 3: Оставшиеся сомы (`remainder = layer_soma_count - sum(allocated)`) распределяются по 1 штуке среди типов нейронов в порядке убывания дробной части `ideal - floor(ideal)`.
   - Шаг 4 (Tie-breaking): При равных дробных частях приоритет отдается типу с меньшим `variant_id` (индексом в массиве `neuron_types` конфигурации шарда).
4. **Инвариант**: Сумма распределенных количеств по всем типам нейронов в слое обязана строго равняться `layer_soma_count`. Превышение емкости слоя `layer_soma_count > layer_capacity` аппаратно невозможно благодаря валидации `density <= 1.0` в `config`.

### §5.3. Стабильный Порядок Назначения Soma ID
Для обеспечения воспроизводимости отладки все сомы нумеруются строго последовательно (`soma_id` от `0` до `total_somas - 1`) в следующем лексикографическом порядке сортировки:
1. Индекс анатомического слоя (`layer_index`);
2. Идентификатор типа нейрона (`variant_id`), который строго равен индексу `NeuronType` в массиве `config::ShardConfig.neuron_types`;
3. Локальный порядковый номер сом внутри данного типа в текущем слое (`local_ordinal`).

### §5.4. Детерминированное Размещение (Deterministic Soma Placement)
Для каждой сомы с уникальным набором `(layer_index, variant_id, local_ordinal)` генерируется координата вокселя по следующему алгоритму:
1. Вычисляется плоская емкость слоя:
   `layer_capacity = shard_w * shard_d * layer_h_voxels` (где `shard_w = dimensions.w`, `shard_d = dimensions.d`).
2. Вычисляется детерминированное стартовое смещение с помощью смесителя хеша:
   `start = deterministic_hash(seed, layer_index, variant_id, local_ordinal) % layer_capacity` (выполняется с помощью побитового смешивания `hash` / `mix` примитивов из `types`).
3. Выполняется детерминированное зондирование по всей емкости слоя (для `attempt` в диапазоне `0..layer_capacity`):
   - Вычисляется плоский индекс вокселя:
     `flat_idx = (start + attempt) % layer_capacity`
   - Индекс преобразуется в трехмерные локальные координаты вокселя:
     `z = z_start + flat_idx / (shard_w * shard_d)`
     `rem = flat_idx % (shard_w * shard_d)`
     `y = rem / shard_w`
     `x = rem % shard_w`
   - Если воксель с координатами `(x, y, z)` свободен, сома размещается в нем, и генерация для данной сомы успешно завершается.
4. Если после перебора всех `layer_capacity` попыток свободный воксель не найден, возвращается ошибка `TopologyError::LayerCapacityExceeded`.
Silent drop (молчаливый пропуск размещения сомы) строго запрещен.

### §5.5. Алгоритмы Роста Аксонов (Stage B1 Local Axon Growth)

Алгоритм детерминированно строит траекторию роста аксона для каждой сомы независимо:

1. **Инициализация**:
   - Для сомы `soma` извлекается ее тип:
     Если `soma.variant_id as usize >= config.neuron_types.len()`, генерация аварийно завершается с ошибкой `TopologyError::UnknownNeuronType { variant_id: soma.variant_id }`.
     Иначе `source_type = &config.neuron_types[soma.variant_id as usize]`.
   - Параметры роста извлекаются из `source_type.growth`.
   - Веса параметров роста конвертируются из `f32` в формат Q16 `i64` строго через валидирующий хелпер:
     - `inertia_q = to_fixed_point_q16(growth.steering_weight_inertia)`
     - `vertical_q = to_fixed_point_q16(growth.growth_vertical_bias)`
     - `jitter_q = to_fixed_point_q16(growth.steering_weight_jitter)`
     Если любое вещественное значение не является конечным (`is_finite()`) или при масштабировании на `65536.0` выходит за диапазон представления `i64`, генерация немедленно возвращает `TopologyError::InvalidGrowthParameter`.
   - Стартовая позиция аксона: воксель сомы `soma.position`.
   - Воксель сомы считается занятым origin-точкой пути (не может повторно посещаться).
   - Номер текущего шага `step_index` инициализируется равным `1`.
   - Предыдущий вектор движения $d_{prev} = (dx_{prev}, dy_{prev}, dz_{prev})$ инициализируется как $(0, 0, 0)$ (первый шаг).

2. **Moore Neighborhood (26 кандидатов) и их упорядочивание**:
   На каждом шаге оцениваются 26 потенциальных направлений движения $u_i = (dx_i, dy_i, dz_i)$, где $dx_i, dy_i, dz_i \in \{-1, 0, 1\}$, исключая $(0, 0, 0)$.
   Кандидаты нумеруются `candidate_index` от `0` до `25` строго в следующем лексикографическом порядке:
   ```rust
   let mut candidate_index = 0;
   for dz in -1..=1 {
       for dy in -1..=1 {
           for dx in -1..=1 {
               if dz == 0 && dy == 0 && dx == 0 {
                   continue;
               }
               // Назначение candidate_index направлению (dx, dy, dz)
               candidate_index += 1;
           }
       }
   }
   ```

3. **Целочисленный расчет Score (Steering Model)**:
   Для каждого из 26 кандидатов вычисляется целочисленный балл `score_q` (в типе `i64`):
   $$score\_q = inertia\_q \cdot \text{dot}(d_{prev}, u_i) + vertical\_q \cdot dz_i + \frac{jitter\_q \cdot jitter\_unit}{65535}$$
   Где:
   - $\text{dot}(d_{prev}, u_i) = dx_{prev} \cdot dx_i + dy_{prev} \cdot dy_i + dz_{prev} \cdot dz_i$ — скалярное произведение векторов, принимающее целые значения от -3 до 3.
   - $dz_i$ — вертикальная координата шага кандидата, принимающая значения от -1 до 1.
   - $jitter\_unit = (\text{seed.random\_u64}(salt) \gg 48) \text{ as } i64$ — 16-битное беззнаковое целое число в диапазоне `0..=65535`, полученное из смесителя:
     `salt = deterministic_mix_salt(soma_id, step_index, candidate_index)`
   - Все умножения, деления и сложения при вычислении `score_q` осуществляются через checked-инструкции в Rust (`checked_mul`, `checked_add`, `checked_div`). Любое арифметическое переполнение немедленно завершает всю генерацию шарда и возвращает ошибку `TopologyError::CapacityOverflow`.
   - Деление `(jitter_q * jitter_unit) / 65535` выполняется как целочисленное деление в Rust (truncating).
   - Все расчеты в горячем цикле роста после прохождения AOT-границы выполняются строго в типе `i64`. Использование вещественной математики (float) строго запрещено.

4. **Сортировка приоритетов кандидатов**:
   Все 26 кандидатов сортируются по следующим критериям:
   - В первую очередь: по значению `score_q` в порядке убывания (descending).
   - Во вторую очередь (tie-break при равенстве score): по значению `candidate_index` в порядке возрастания (ascending).

5. **Выбор направления и правила препятствий / границ (Stop Reasons)**:
   Алгоритм последовательно перебирает отсортированный список кандидатов:
   - **Условие Выхода за Границы**: Если текущий кандидат $u_i$ ведет в воксель вне геометрических границ шарда ($x_{new} \notin [0..w)$ или $y_{new} \notin [0..d)$ or $z_{new} \notin [0..h)$):
     Рост текущего аксона немедленно останавливается с причиной `AxonGrowthStopReason::BoundaryReached`. Данный кандидат не добавляется в segments. Перебор прекращается.
   - **Условие Препятствия (Obstacle)**: Если воксель кандидата совпадает с позицией сомы любого другого нейрона шарда, данный кандидат пропускается (алгоритм переходит к следующему в списке кандидатов).
   - **Условие Самопересечения (Self-Intersection)**: Если воксель кандидата совпадает с позицией исходной сомы или любого ранее созданного сегмента этого же аксона, данный кандидат пропускается.
   - **Успешный Шаг**: Первый встреченный кандидат, не вызвавший остановку по границе и не заблокированный препятствиями/самопересечением, выбирается в качестве следующего сегмента.

6. **Добавление сегмента и шаг цикла**:
   - Создается `AxonSegment` с позицией, полученной через `PackedPosition::try_new(x, y, z, source_variant_id)`. Таким образом, тип вокселей сегментов аксона совпадает с `variant_id` исходной сомы.
   - `segment_offset` нового сегмента устанавливается равным текущему `step_index`.
   - Предыдущий вектор движения обновляется: $d_{prev} = P_{new} - P_{curr}$.
   - Выполняется переход к шагу `step_index + 1`.

7. **Условия остановки роста**:
   Рост аксона прекращается, если выполнено одно из условий:
   - **MaxLengthReached**: `step_index` достиг значения `min(layout::MAX_SEGMENTS_PER_AXON - 1, types::MAX_SEGMENT_OFFSET as usize)` (при значениях констант 256 и 255 предел равен 255).
   - **BoundaryReached**: В процессе перебора кандидатов выбран кандидат, ведущий за границы шарда.
   - **Blocked**: Весь список из 26 кандидатов перебран, при этом не встречен out-of-bounds кандидат, и все in-bounds кандидаты отсеяны как препятствия или самопересечения.

## §6. Побитовый Детерминизм (Determinism Policy)

1. Категорически запрещено использование системного времени (`SystemTime`), адресов памяти указателей и недетерминированных хэш-таблиц.
2. Весь генеративный пайплайн опирается исключительно на смесители на базе `MasterSeed` и входных параметров DTO.

## §8. Требуемые Инварианты (Stage A & Stage B1)

- **INV-TOPO-001**: *Ограничение плотности вокселей*. В одном вокселе трехмерной сетки шарда может находиться не более одной сомы нейрона.
- **INV-TOPO-002**: *Контроль емкости слоев*. Если физическое место для размещения требуемого количества сом исчерпано, возвращается `TopologyError::LayerCapacityExceeded`.
- **INV-TOPO-003**: *Соответствие типов*. Поле `variant_id` в `PlacedSoma` строго соответствует индексу `NeuronType` в массиве `ShardConfig.neuron_types`.
- **INV-TOPO-004**: *Детерминизм вывода*. Один и тот же набор `SingleShardTopologyInput` обязан возвращать побитово идентичный результат `SingleShardTopology` на любых поддерживаемых архитектурах.
- **INV-TOPO-B1-001**: *Детерминизм траектории*. При одинаковых входных данных путь роста аксона полностью воспроизводим.
- **INV-TOPO-B1-002**: *Лимит длины*. Количество сегментов пути аксона не превышает `min(layout::MAX_SEGMENTS_PER_AXON - 1, types::MAX_SEGMENT_OFFSET as usize)` (т.е. 255 сегментов), а смещение сегментов (`segment_offset`) строго возрастает по порядку от `1` до длины пути.
- **INV-TOPO-B1-003**: *Локальность*. Все сегменты аксона находятся строго в границах шарда `[0..w) x [0..d) x [0..h)`.
- **INV-TOPO-B1-004**: *Запрет самопересечений*. Путь аксона не содержит дубликатов вокселей (аксон не пересекает сам себя и не возвращается на воксель исходной сомы).
- **INV-TOPO-B1-005**: *Фильтрация препятствий*. Ни один сегмент аксона не может занимать воксель, занятый сомой любого другого нейрона.
- **INV-TOPO-B1-006**: *Отсутствие побочных эффектов*. Stage B1 выполняет только трассировку геометрии путей аксонов в памяти и не генерирует синапсы или SoA layout записи.
- **INV-TOPO-B1-007**: *Сохранение типов*. Координата вокселя каждого сегмента аксона строго удовлетворяет условию: `segment.position.type_id() == source_soma.variant_id`.
- **INV-TOPO-B1-008**: *Сохранение порядка вывода*. Список выращенных путей `LocalGrowthResult.axons` сохраняет взаимно-однозначное соответствие по длине и порядку следования элементам из входного вектора `input.topology.somas`.

## §9. Golden Tests / Обязательная Матрица Тестирования (Stage A & Stage B1)

Реализация Stage A и Stage B1 должна быть полностью покрыта следующими тестами:

### Тесты этапа Stage A
1. **Воспроизводимость размещения сом (`test_topology_reproducible_soma_placement`)**: Верификация 100% совпадения координат сом при повторных запусках с одинаковым `MasterSeed`.
2. **Детерминизм изменения семени (`test_topology_seed_changes_placement_but_preserves_counts`)**: Проверка, что другое семя `MasterSeed` меняет позиции сом, но оставляет неизменным их число по типам.
3. **Отсутствие коллизий вокселей (`test_topology_no_voxel_collisions`)**: Проверка инварианта `INV-TOPO-001` — не более одной сомы на воксель.
4. **Соблюдение плотности слоев (`test_topology_layer_density_respected`)**: Проверка соответствия числа размещенных нейронов расчетной плотности.
5. **Детерминизм Largest-Remainder (`test_topology_composition_largest_remainder_deterministic`)**: Верификация математики Гамильтона распределения остатка.
6. **Валидация границ PackedPosition (`test_topology_packed_position_bounds`)**: Проверка укладывания сгенерированных вокселей в диапазоны Z-слоев.
7. **Браковка слоев нулевой высоты (`test_topology_zero_height_layer_rejected`)**: Возврат ошибки `LayerGeometryError`, если слой получает 0 вокселей высоты при плотности $> 0.0$.
8. **Полное заполнение слоя (`test_topology_full_layer_capacity_fill`)**: Проверка, что при `density = 1.0` на небольшом слое алгоритм успешно и без ошибок заполняет все доступные воксели (100% емкость) без коллизий.
8a. **Защитная проверка превышения емкости (`test_topology_layer_capacity_exceeded_defensive`)**: Тестирование на внутреннем/вспомогательном уровне, проверяющее возврат ошибки `LayerCapacityExceeded` при попытке разместить больше сом, чем воксельная емкость слоя. Данное состояние недостижимо снаружи благодаря валидации конфигурации (`density <= 1.0`), но проверяется как защитный инвариант.
9. **Детерминированное зондирование при коллизиях (`test_topology_deterministic_collision_probing`)**: Проверка прохождения шагов зондирования при искусственной коллизии кандидатов.
10. **Стабильный порядок Soma ID (`test_topology_stable_soma_id_ordering`)**: Проверка упорядочивания `soma_id` по правилу `layer -> variant -> local_ordinal`.
11. **Изоляция зависимостей (`test_topology_no_forbidden_dependencies`)**: Статический контроль отсутствия связей с `compute`, `wire`, `ipc`, `vfs`.

### Тесты этапа Stage B1
12. **Целочисленный расчет направления (`test_topology_fixed_point_steering_no_float_runtime`)**: Верификация того, что в процессе пошагового роста выбор направления Moore neighborhood дает точный fixed-point результат без привлечения вещественного рантайма.
13. **Использование параметров роста типа нейрона (`test_topology_growth_uses_source_variant_growth_params`)**: Проверка, что рост каждого аксона использует индивидуальные `GrowthParams` из соответствующего `neuron_types[variant_id]`.
14. **Сохранение type_id в координатах сегментов (`test_topology_segment_position_type_id_matches_source_variant`)**: Проверка, что каждый созданный `AxonSegment` содержит упакованную координату с `type_id`, в точности равным `variant_id` исходной сомы.
15. **Браковка неизвестного типа нейрона (`test_topology_unknown_variant_id_rejected`)**: Верификация возврата ошибки `TopologyError::UnknownNeuronType` при обнаружении `variant_id`, выходящего за пределы диапазона `neuron_types`.
16. **Детерминированная остановка на границе после выбора OOB-вектора (`test_topology_boundary_stop_reason_after_selected_oob_direction`)**: Проверка, что если кандидат с наибольшим приоритетом (score) указывает за границы шарда, рост останавливается с `BoundaryReached`, даже если есть альтернативные незаблокированные in-bounds кандидаты с меньшим score.
17. **Запрет возврата в исходную сому (`test_topology_source_origin_not_revisited`)**: Проверка, что ни один шаг пути аксона не возвращается в исходную координату сомы (origin считается препятствием для самого себя).
18. **Согласованность констант лимитов (`test_topology_layout_types_segment_limit_consistency`)**: Проверка совпадения значений максимального количества сегментов на этапе компиляции: `layout::MAX_SEGMENTS_PER_AXON - 1 == types::MAX_SEGMENT_OFFSET` (255).
19. **Воспроизводимость путей роста (`test_topology_reproducible_growth_path`)**: Верификация 100% идентичности сгенерированных сегментов аксона при повторной трассировке с одинаковым `MasterSeed`.
20. **Диагностический CSV роста (`test_topology_probe_growth_csv_format`)**: Верификация формата генерируемого диагностического CSV файла для путей аксонов: `soma_id,segment_offset,x,y,z`.
21. **Ограничение максимальной длины аксона (`test_topology_max_segment_length_respected`)**: Проверка, что длина пути аксона не превышает лимит в 255 сегментов, и в случае достижения лимита устанавливается причина остановки `MaxLengthReached`.
22. **Непрерывность смещений сегментов (`test_topology_segment_offset_contract`)**: Проверка, что смещение `segment_offset` для сегментов одного аксона увеличивается монотонно и последовательно без пропусков (начиная с 1).
23. **Начало роста вне границ шарда (`test_topology_source_out_of_bounds_stop_reason`)**: Проверка, что если сома нейрона расположена за границами воксельного пространства шарда, то путь аксона пуст, а причина остановки устанавливается в `SourceOutOfBounds`.
24. **Браковка некорректных параметров роста (`test_topology_invalid_growth_nan_rejected`)**: Проверка возврата ошибки `TopologyError::InvalidGrowthParameter` на этапе Q16-конверсии при обнаружении NaN или бесконечных величин в параметрах роста.
25. **Защита от переполнения целочисленного score (`test_topology_score_overflow_rejected`)**: Проверка, что если параметры роста чрезвычайно велики (но проходят проверку на конечность), то при расчете `score_q` арифметическое переполнение отлавливается и возвращается ошибка `TopologyError::CapacityOverflow`.
