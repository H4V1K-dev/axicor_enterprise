## §3. Инварианты

### §3.1. Структурные инварианты

- **INV-BAKER-001**: *Делегирование полномочий (Zero-Duplication)*.
  - *Обоснование*: Крейт `baker` является функциональным оркестратором и конвейером сборки. Он **не имеет права** дублировать математику, константы или логику выравнивания. Выравнивание страниц ОС делегировано `vfs`, 3D-математика — `topology`, физиологические формулы — `physics`, а C-ABI макеты памяти — `layout`.
  - *Следствие нарушения*: Рассинхронизация алгоритмов между AOT-компилятором и рантаймом GPU, Silent Data Corruption при загрузке `mmap`.
  - *Где проверяется*: compile-time (архитектурный линтер и структура зависимостей).

### §3.2. Семантические инварианты

- **INV-BAKER-002**: *Строгий порядок конвейера (Pipeline Order Invariant)*.
  - *Обоснование*: Сборка графа не может выполняться в произвольном порядке из-за жестких data dependencies. Фаза B (генерация локальной геометрии и выходных матриц `.gxo`) обязана завершиться **до** старта Фазы C (макро-маршрутизация `.ghosts`), так как маршрутизатору требуются уже аллоцированные `soma_id` целевых нейронов.
  - *Следствие нарушения*: Запись мусорных или неинициализированных `soma_id` в файл `.ghosts`, крах межшардовой маршрутизации в рантайме.
  - *Где проверяется*: runtime assert в `pipeline.rs`.

- **INV-BAKER-003**: *Абсолютный детерминизм (Bit-to-Bit Reproducibility)*.
  - *Обоснование*: При фиксированном `master_seed`, компилятор обязан генерировать побитово идентичные выходные бинарники (`.state`, `.axons`, `.ghosts`, `.paths`) на любой ОС и архитектуре (x86/ARM). Использование системного времени (`SystemTime::now`), недетерминированных хэш-мап (стандартных `HashMap` без детерминированного хэшера) или неконтролируемого параллелизма запрещено.
  - *Следствие нарушения*: Дивергенция распределенных нод в кластере, невозможность отладки гейзенбагов.
  - *Где проверяется*: интеграционные тесты `test_pipeline_reproducibility`.

- **INV-BAKER-004**: *Гарантия Fail-Fast валидации (Pre-Bake Guard)*.
  - *Обоснование*: `baker` не пытается "исправить" некорректные конфиги. Если AST-деревья из `config` содержат нарушения макро-квот (например, сумма `height_pct` слоев != 1.0, или рассчитанная скорость `v_seg` имеет дробный остаток), компилятор обязан мгновенно прервать работу и вернуть `Err` до начала тяжелых $O(N)$ вычислений и выделения памяти в хосте.
  - *Следствие нарушения*: Генерация поврежденных `.axic` архивов, приводящих к аппаратному `panic!` внутри HFT-цикла GPU.
  - *Где проверяется*: `validator.rs` на Фазе A до запуска геометрии.

### §3.3. Межкрейтовые инварианты

Секция не применима к данному крейту: `baker` является абсолютным потребителем инвариантов из нижележащих слоев (таких как `INV-CROSS-010` из `layout` или `INV-TOPO-001` из `topology`) и не объявляет собственных межкрейтовых контрактов, требующих встречной верификации от других крейтов.

## §4. Публичный API

Крейт имеет минимальную площадь контакта. Он не хранит состояние в памяти хоста дольше времени сборки графа и предоставляет единственную точку входа для запуска всего сборочного конвейера.

### §4.1. Типы

#### AxicArchive

```rust
pub struct AxicArchive {
    // Внутренние поля скрыты
}
```

- **Семантика:** Скомпилированный в памяти архив симуляции.
- **Жизненный цикл:** Инициализируется в конце Baking pipeline. Делегирует свою физическую упаковку и сериализацию на диск структуре `AxicPacker` из крейта `vfs`.

### §4.2. Трейты

Секция не применима к данному крейту: публичные полиморфные трейты отсутствуют.

### §4.3. Функции

```rust
pub fn bake(config_dir: &Path) -> Result<AxicArchive, BakerError>
```

- **Назначение:** Оркестрирует процесс сборки TOML-конфигураций в бинарные файлы (Фазы A → E).
- **Предусловия:** Указанный каталог содержит валидные TOML-конфигурации (`simulation.toml`, `anatomy.toml`, `blueprints.toml`, `brain.toml`).
- **Постусловия:** Возвращает структуру `AxicArchive`, смещения бинарных файлов в которой аппаратно выровнены по 4096 байт для последующего Zero-Copy маппинга.
- **Сложность:** O(N) по времени, где N — общее количество нейронов во всех слоях шарда.
- **Паника:** Никогда. В случае сбоя I/O или нарушения архитектурных квот (например, нехватка слотов), делает жесткий Fail-Fast с возвратом `Err(BakerError)`.

### §4.4. Константы и Магические Числа

Секция не применима к данному крейту.

В соответствии с инвариантом INV-BAKER-001 (Zero-Duplication), `baker` не объявляет собственных магических чисел. Все константы (размер страницы ОС `OS_PAGE_SIZE`, маркер отсутствия сомы `EMPTY_PIXEL`, пороги `BAKE_MAGIC`) жестко импортируются из профильных контрактных крейтов (`vfs`, `types`, `wire`).

## §5. Доменная Логика

Крейт `baker` — это AOT-компилятор (Ahead-of-Time) нейроморфного графа. Это оффлайн-инструмент сборочного конвейера, а не часть горячего рантайма. Его единственная доменная задача — переварить декларативные человекочитаемые конфигурации (TOML) и сгенерировать побитово-детерминированные, выровненные по кэш-линиям L2 бинарные структуры (SoA массивы в формате C-ABI) для мгновенной (Zero-Copy) загрузки в VRAM видеокарты.

В памяти компилятора не создается объектов и разветвленных графов на указателях. Сборка осуществляется через строгий однонаправленный конвейер (Phases A→B→C→D→E), где выход одной фазы является неизменяемым входом для следующей.

### §5.1. Фаза A: Деривация биологических констант (Biological Derivation)

Трансляция "человеческих" физических единиц измерения (мс, мкм, Гц, м/с) в жесткие дискретные аппаратные кванты (тики, воксели, фазовые аккумуляторы). Выполняется до начала генерации графа для заполнения `VariantParameters` (которая будет загружена в Constant Memory GPU) и расчета глобальных физических переменных.

- **Синтез DDS-пейсмейкера**: `spontaneous_firing_period_ticks` пересчитывается в 16-битный фазовый аккумулятор `heartbeat_m` (`65536 / period_ticks`). Это позволяет GPU-ядру вычислять спонтанную активность без хранения состояний таймеров для каждого нейрона.
- **Дискретизация скорости**: `signal_speed_m_s`, `voxel_size_um` и `tick_duration_us` транслируются в `v_seg` (число сегментов воксельной сетки, проходимых сигналом за 1 тик). 
- **Модуляторы R-STDP**: Флаг `is_inhibitory` преобразуется в аффинности дофаминовых рецепторов (`d1_affinity`, `d2_affinity`), аппаратно задавая чувствительность конкретного типа нейрона к глобальному сигналу награды (LTP-усилитель для возбуждающих, LTD-модулятор для тормозных).

### §5.2. Фаза B: Пространственная генерация (Spatial Generation)

Основная задача фазы — трансляция слоистой анатомии (`AnatomyConfig`) и I/O матриц в физические координаты (воксели). `baker` не реализует 3D-математику самостоятельно, а жестко оркестрирует вызовы к O(1) и O(N) алгоритмам крейта `topology`:
- **Размещение сом**: Через `topology::place_somas` генерируется плоский массив `PackedPosition`. Строго соблюдается квотирование Bottom-Up, полностью исключающее потерю нейронов при float-дрифте плотности.
- **Виртуализация входов**: Для каждой матрицы сенсоров из `IoConfig` вызывается `topology::generate_virtual_axons`, прошивающая шард сквозными Z-колоннами. Применяется детерминированный UV-джиттер для предотвращения hotspot-коллизий в вокселях.
- **Трассировка локальных путей**: Запускается параллельный `topology::cone_tracing` для всех локальных аксонов. Сгенерированные сегменты (Step-and-Pack) накапливаются в промежуточных буферах хоста для последующего дампа в `.paths`.

### §5.3. Фаза C: Макро-маршрутизация (Ghost Atlas Routing)

Сборка статической таблицы межзональных связей (`.ghosts`). Согласно инварианту `INV-BAKER-002` (Pipeline Order), эта фаза обязана стартовать строго после завершения Фазы B, так как для связывания требуются уже аллоцированные `soma_id` из выходных матриц.
- **Резолв соединений**: `baker` обходит массив макро-связей из `brain.toml` и для каждой пары (from -> to) вызывает `topology::route_ghost_atlas`.
- **Zero-Cost Абстракция**: В результате формируется плоский C-ABI массив `GhostConnection` (`src_soma_id` → `target_ghost_id`). Эта абстракция позволяет GPU-ядру рантайма перебрасывать спайки между зонами за O(1), не имея ни малейшего понятия о 3D-геометрии, границах шардов или сети.
- **Аппаратные заглушки**: Если маршрутизатор не находит подходящей сомы в целевом регионе, он аппаратно прописывает `EMPTY_PIXEL` (0xFFFF_FFFF). В горячем цикле это вызывает мгновенный `Early Exit` потока варпа.

### §5.4. Фаза D: Формирование Memory Layout (SoA Alignment & Quantization)

На этом этапе `baker` берет сгенерированные координаты и связи, чтобы уложить их в GPU-оптимизированные форматы. Никаких деревьев и объектов — только плоские C-ABI структуры из крейта `layout`.
- **Z-Sort (Cache Locality)**: Все нейроны принудительно сортируются по оси Z (слоям). Это гарантирует, что физически близкие нейроны будут лежать рядом в памяти, обеспечивая 100% попадание (Coalesced Access) в L2 кэш GPU при пространственных выборках.
- **Warp Alignment**: Количество нейронов аппаратно выравнивается (`padded_n`) вверх до ближайшего кратного 64 (размер кэш-линии и Wavefront). Пустые слоты заполняются маркерами `AXON_SENTINEL`.
- **Бинарные дампы**: Формируются плоские массивы для файлов `.state` (SoA-вектора вольтажа, весов, порогов), `.axons` (сдвиговые регистры `BurstHeads8`) и `.paths` (квантованные в `u32` 3D-координаты сегментов).

### §5.5. Фаза E: Архивация (VFS Container Packaging)

Оффлайн-сборка графа завершается созданием единого переносимого ROM-контейнера (`.axic`). `baker` не реализует логику файловой системы сам, а строго делегирует её крейту `vfs` (Слой 2).
- **Zero-Copy Readiness**: Все сгенерированные файлы (`.state`, `.axons`, `.paths`, `.gxi`, `.gxo`, `.ghosts`) передаются упаковщику `AxicPacker`.
- **OS Page Alignment**: Упаковщик безусловно выравнивает смещение каждого вложенного файла по границе 4096 байт (стандартный размер страницы Linux/Windows). Это аппаратное требование позволяет рантайму в будущем сделать `mmap` любого файла напрямую из архива в виртуальную память, минуя копирование в User-Space (Zero-Copy Mmap).

## §6. Алгоритмы и Формулы

В соответствии с инвариантом **INV-BAKER-001** (Zero-Duplication), крейт `baker` не содержит собственной математики и формул. Вся доменная физика (пересчет тиков в фазовый аккумулятор DDS, расчет `v_seg` и аффинностей дофамина D1/D2) делегируется чистым `const fn` функциям из крейта `physics` (Слой 0). 

Единственный алгоритм крейта `baker` — это строгая топологическая сортировка графа сборки.

### §6.1. Топологическая сортировка конвейера (Pipeline Dependency Resolution)

- **Вход**: Ссылки на валидированные AST-деревья из крейта `config` (`SimulationConfig`, `BrainConfig`).
- **Выход**: Сформированный `AxicArchive`.
- **Детерминизм**: Да.

**Логика разрешения зависимостей (Topological Sort):**

Сборка графа не может выполняться в произвольном порядке из-за жестких data dependencies между фазами. Алгоритм компилятора гарантирует следующую последовательность блокировок:

1. **Биологическая деривация (Фаза A)**: Выполняется первой. Извлекает `signal_speed_m_s` и вызывает `physics::compute_v_seg()`. При возврате `None` (дробный шаг) компиляция аппаратно прерывается.
2. **Генерация I/O (Фаза B - Входы)**: Вызывает `topology::generate_virtual_axons()` для входных матриц. Резервирует слоты `axon_heads`.
3. **Размещение сом (Фаза B - Геометрия)**: Вызывает `topology::place_somas()`. Массив `PackedPosition` блокируется на запись.
4. **Трассировка и Выходы (Фаза B - I/O)**: Локальные аксоны вызывают `topology::cone_tracing()`. Массивы выходных матриц собирают сгенерированные `soma_id`.
5. **Макро-маршрутизация (Фаза C)**: Ожидает завершения пунктов 3 и 4. Использует собранные `soma_id` для вызова `topology::route_ghost_atlas()` и генерации `.ghosts`.
6. **Выравнивание (Фаза D)**: Ожидает завершения пункта 3. Запускает `align_to_warp()` из крейта `layout`.
7. **Упаковка (Фаза E)**: Ожидает завершения всех предыдущих фаз. Передает все бинарные блобы в `vfs::AxicPacker`.

Если любая из фаз возвращает `Err`, весь конвейер немедленно останавливается (Fail-Fast), предотвращая генерацию частично битых `.axic` архивов.

## §8. Граничные Случаи и Особые Сценарии

Крейт `baker` работает в режиме AOT-компиляции. Его главная архитектурная задача в контексте граничных случаев — **Fail-Fast** (мгновенная остановка). Компилятор аппаратно не поддерживает "частичную сборку" или "деградацию", чтобы исключить попадание поврежденных C-ABI структур в Zero-Copy `.axic` архив.

### §8.1. Граничные значения

| # | Ситуация | Ожидаемое поведение |
|---|----------|-------------------|
| E-086 | Период спонтанных спайков `period == 0` | `heartbeat_m` устанавливается в 0 (аппаратное отключение DDS-пейсмейкера для данного типа нейронов без ветвлений). |
| E-087 | Нецелочисленная скорость сигнала (`compute_v_seg` дает остаток) | Fail-Fast с возвратом ошибки `BakerError::InvalidSignalSpeed`. Защищает целочисленную физику (Integer Physics) от FPU-дрифта в рантайме. |
| E-088 | Сумма `height_pct` в анатомии не равна 1.0 | Fail-Fast с возвратом ошибки `BakerError::InvalidLayerHeights`. Предотвращает появление неинициализированных "дыр" при `Z-Sort` упаковке в память. |
| E-089 | Сумма долей `composition` в слое не равна 1.0 | Fail-Fast с возвратом ошибки `BakerError::InvalidComposition`. Защищает `Reject-Sampling` пулы от бесконечного цикла при генерации шарда. |
| E-090 | Входной каталог TOML пуст или отсутствует | Fail-Fast с возвратом ошибки `BakerError::ConfigNotFound`. |
| E-091 | Превышение лимита межшардовых связей при роутинге атласа | В Фазе C курсор Ghost-аксонов выходит за пределы `ghost_capacity_limit`. Выполняется аппаратный `assert!`, предотвращающий переполнение буфера VRAM (Cursor Boundary Assert). |

### §8.2. Состояния гонки и конкурентность

Секция не применима к данному крейту. 
Компиляция графа разделена на строгие топологические фазы (INV-BAKER-002). Внутри фаз используется чистый функциональный Map-Reduce через `rayon`. Отсутствие разделяемого мутабельного состояния (`Arc<Mutex<T>>`) делает Data Races математически невозможными.

### §8.3. Деградация и Recovery

Секция не применима к данному крейту. 
Компилятор не поддерживает деградацию или восстановление. При возникновении любой ошибки валидации или нехватки ресурсов выполняется жесткий Fail-Fast (сборка останавливается с возвратом `Err`), чтобы гарантировать, что сгенерированный `.axic` архив на 100% безопасен для Zero-Copy загрузки в GPU.

## §9. Ошибки

В отличие от рантайма, AOT-компилятор `baker` не пытается "переваривать" сбои. Здесь применяется строгая политика Fail-Fast.

### §9.1. Перечисление ошибок

```rust
#[derive(Debug)]
pub enum BakerError {
    /// Входной каталог конфигураций не найден (E-090)
    ConfigNotFound { path: String },
    /// Неверная высота слоев: сумма не равна 1.0 (E-088)
    InvalidLayerHeights { sum: f32 },
    /// Невалидный состав нейронов в слое: сумма не равна 1.0 (E-089)
    InvalidComposition { layer: String, sum: f32 },
    /// Скорость сигнала не делится нацело на квант длины (E-087)
    InvalidSignalSpeed { speed: f32, step_size: f32 },
    /// Ошибка ввода-вывода при записи .axic архива
    IOError(String),
}
```

### §9.2. Стратегия обработки

| Ошибка | Восстановимая? | Рекомендация вызывающему |
|--------|----------------|--------------------------|
| `ConfigNotFound` | Нет | Проверить CLI-аргументы и наличие файлов |
| `InvalidLayerHeights` | Нет | Изменить `height_pct` в `anatomy.toml`, чтобы сумма была строго 1.0 |
| `InvalidComposition` | Нет | Изменить доли `composition` в `anatomy.toml`, чтобы сумма была строго 1.0 |
| `InvalidSignalSpeed` | Нет | Подогнать `signal_speed_m_s` или `segment_length_voxels` в `simulation.toml` |
| `IOError` | Нет | Проверить права доступа и свободное место на диске |

### §9.3. Паники

| Условие | Почему паника, а не Err |
|---------|------------------------|
| `assert!(*cursor <= max_allowed)` при Ghost-роутинге (E-091) | Нарушение аппаратной квоты `ghost_capacity` для шарда-получателя. Предотвращает переполнение буфера VRAM (Cursor Boundary Assert). Продолжение сборки приведёт к Silent Data Corruption при загрузке. |
