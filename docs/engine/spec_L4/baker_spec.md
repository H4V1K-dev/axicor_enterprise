|---|---|
| Название | `baker` |
| Слой | Слой 4 — Topology, Baker & Edge Conversion |
| Тип | Library (lib) |

## §3. Инварианты

### §3.1. Структурные инварианты

- **INV-BAKER-001**: *Делегирование полномочий (Zero-Duplication)*.
  - *Обоснование*: Крейт `baker` является функциональным оркестратором. Он не должен дублировать математику, константы или логику выравнивания. Выравнивание страниц ОС делегировано `vfs`, 3D-математика — `topology`, физиологические формулы — `physics`, а C-ABI макеты памяти — `layout`.
  - *Следствие нарушения*: Рассинхронизация алгоритмов компиляции и исполнения, Silent Data Corruption.
  - *Где проверяется*: compile-time (структура импортов и зависимостей в Cargo.toml).

### §3.2. Семантические инварианты

- **INV-BAKER-002**: *Строгий порядок конвейера (Pipeline Order Invariant)*.
  - *Обоснование*: Сборка графа не может выполняться в произвольном порядке. Фаза B (генерация локальной геометрии) обязана завершиться до старта Фазы C (макро-маршрутизация), так как маршрутизатору требуются уже готовые сомы целевых нейронов.
  - *Следствие нарушения*: Запись неинициализированных `soma_id` в файл `.ghosts`.
  - *Где проверяется*: runtime порядок вызовов в `pipeline.rs`.

- **INV-BAKER-003**: *Абсолютный детерминизм (Bit-to-Bit Reproducibility)*.
  - *Обоснование*: При фиксированном `master_seed`, компилятор обязан генерировать побитово идентичные выходные файлы архива на любой ОС и архитектуре. Использование системного времени, недетерминированных хэшеров или неконтролируемого параллелизма запрещено.
  - *Следствие нарушения*: Дивергенция распределенных нод, невозможность отладки гейзенбагов.
  - *Где проверяется*: интеграционные тесты `test_pipeline_reproducibility`.

- **INV-BAKER-004**: *Гарантия Fail-Fast валидации (Pre-Bake Guard)*.
  - *Обоснование*: `baker` не пытается исправить некорректные конфиги. Любые ошибки структуры (например, сумма `height_pct` слоев != 1.0, дробный шаг скорости) вызывают немедленный возврат `Err` до начала тяжелых вычислений и выделения памяти.
  - *Следствие нарушения*: Генерация поврежденных `.axic` архивов.
  - *Где проверяется*: `validator::validate_physics_and_anatomy` и Serde-валидаторы на Фазе A.

## §4. Публичный API

Крейт предоставляет единственную точку входа для запуска сборочного конвейера.

### §4.1. Типы

#### `AxicArchive`
```rust
pub struct AxicArchive {}
```
- **Семантика:** Скомпилированный в памяти архив симуляции шарда.

---

### §4.3. Функции

#### `pub fn bake(config_dir: &std::path::Path, output_path: &std::path::Path) -> Result<AxicArchive, BakerError>`
- **Назначение:** Оркестрирует процесс сборки TOML-конфигураций в бинарные файлы и пакует их в `.axic` архив (Фазы A → E).
- **Предусловия:** Указанный каталог содержит валидные TOML-файлы (`model.toml` и `shard.toml`).
- **Постусловия:** Записывает постранично выровненный `.axic` архив по указанному пути.
- **Паника:** Никогда.

## §5. Доменная Логика

Крейт `baker` — это Ahead-of-Time компилятор нейроморфного графа. Его задача — переварить декларативные TOML-файлы и сгенерировать побитово-детерминированные, выровненные по кэш-линиям бинарные SoA структуры (дампы `.state`, `.axons`, `.paths`) для мгновенной Zero-Copy загрузки в память.

Сборка осуществляется через строгий однонаправленный конвейер:
- **Phase A**: Деривация биологических констант и валидация физики/анатомии (`validate_physics_and_anatomy`).
- **Phase B**: Размещение сом в пространстве шарда (`place_somas`).
- **Phase C**: Макро-маршрутизация Ghost-связей через сокеты шарда (`route_ghost_atlas`).
- **Phase D**: Упаковка SoA-массивов (`StateFileHeader`, `AxonsFileHeader`, `PathsFileHeader`).
- **Phase E**: Физическая упаковка временного каталога в `.axic` архив (`pack_directory`).

## §8. Граничные Случаи и Особые Сценарии

### §8.1. Граничные значения

| # | Ситуация | Ожидаемое поведение |
|---|----------|-------------------|
| E-086 | Период спонтанных спайков `period == 0` | DDS-аккумулятор `heartbeat_m` равен 0 (пейсмейкер отключен). |
| E-087 | Нецелочисленная скорость сигнала | Возврат `BakerError::InvalidSignalSpeed` (INV-CONFIG-003). |
| E-088 | Сумма `height_pct` слоев не равна 1.0 | Возврат `BakerError::InvalidLayerHeights` (INV-BAKER-001). |
| E-089 | Сумма долей в слое не равна 1.0 | Возврат `BakerError::InvalidComposition` (INV-BAKER-001). |
| E-090 | Входной каталог пуст или отсутствует | Возврат `BakerError::ConfigNotFound`. |
| E-091 | Превышение лимита `ghost_capacity` | Паника `assert!` при Ghost-роутинге (защита VRAM получателя). |

## §9. Ошибки

### §9.1. Перечисление ошибок

```rust
#[derive(Debug)]
pub enum BakerError {
    /// Требуемый конфигурационный файл не найден
    ConfigNotFound(std::path::PathBuf),
    /// Сумма высот слоев не равна 1.0
    InvalidLayerHeights { actual_sum: f32 },
    /// Сумма долей нейронов в слое не равна 1.0
    InvalidComposition { layer_name: String, actual_sum: f32 },
    /// Параметры скорости сигнала нарушают целочисленную физику
    InvalidSignalSpeed(String),
    /// Ошибка файлового ввода-вывода
    IOError(std::io::Error),
}
```
